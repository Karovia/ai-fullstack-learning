# 第 09 章 · 毕业项目（8-12 周）

> 不是练手作业，是**真正能上线、能赚钱、能写进简历的产品**。

## 🎯 章节目标

- 用前 8 章所有技能做一个**完整产品**
- 走完产品全生命周期：用户调研 → 设计 → 开发 → 测试 → 部署 → 运营 → 迭代
- 至少 100 个真实用户使用
- 写一份能用 5 分钟讲清楚的简历项目卡片
- 在 GitHub 拿到第一批 star（目标 100+）

## 🌱 选题原则

### ✅ 好选题
- 解决你**自己**真实痛点（你最了解需求）
- 有清晰的 AI Agent 价值（不是套壳）
- 6 周能做出 MVP
- 有可能商业化（不强求）
- 能展示前端 + 后端 + AI 三种能力

### ❌ 差选题
- "下一代 ChatGPT"（太大，做不出）
- 纯前端无 AI（浪费 7-8 章学的）
- 别人已经做得很好（差异化在哪？）

## 💡 11 个推荐方向（选 1）

### 方向 A：垂直行业 Agent
1. **[[毕业项目-候选/A1-法律合同审查Agent]]** - 上传合同 → AI 标红风险条款 → 给出修改建议
2. **[[毕业项目-候选/A2-医学论文助理]]** - PubMed 检索 + 综述生成 + 参考文献管理
3. **[[毕业项目-候选/A3-中小学题目讲解Agent]]** - 拍照 → 识别 → 多步讲解 + 知识点延伸

### 方向 B：开发者工具
4. **[[毕业项目-候选/B1-PR代码审查Agent]]** - GitHub App，自动审 PR + 给评论
5. **[[毕业项目-候选/B2-API文档自动生成]]** - 喂代码库 → 出 OpenAPI + 使用示例
6. **[[毕业项目-候选/B3-本地化全自动翻译]]** - 多文件项目 i18n 一键翻译，保持上下文一致

### 方向 C：内容生产
7. **[[毕业项目-候选/C1-个人知识助手]]** - 类似 NotebookLM，针对自己的 Obsidian Vault
8. **[[毕业项目-候选/C2-小红书爆款洗稿器]]** - 给主题 → 输出 5 个不同角度爆款标题 + 图文（合规版）
9. **[[毕业项目-候选/C3-播客转博客]]** - 上传播客 → 转写 → 摘要 → SEO 博客

### 方向 D：自动化代理
10. **[[毕业项目-候选/D1-工作流自动化平台]]** - n8n + LLM 简版，可视化编排
11. **[[毕业项目-候选/D2-个人 OS]]** - 邮件 / 日历 / 待办 / 财务 全部 Agent 化

## 📅 12 周路线（以方向 C1 个人知识助手为例）

### 阶段 1：调研 + 设计（1-2 周）
- W1：用户调研（5 个目标用户访谈）
- W1：竞品分析（NotebookLM / Mem / Reflect）
- W2：产品设计（Figma 画 10 个主页面）
- W2：技术选型 + 架构设计（写 RFC 文档）

### 阶段 2：MVP（3-6 周）
- W3：搭基础架构（Monorepo + CI/CD）
- W3-4：后端 + 数据库 + 认证
- W4-5：前端 UI + 核心功能
- W5-6：AI 接入 + RAG + Agent
- W6：联调 + bug 修

### 阶段 3：发布 + 运营（7-9 周）
- W7：内测（邀请 20 个用户用 1 周）
- W7：根据反馈快速迭代
- W8：正式发布（ProductHunt / V2EX / 即刻 / 小红书）
- W9：埋点 + 数据分析 + 持续优化

### 阶段 4：商业化（10-12 周，可选）
- W10：付费墙 + Stripe 集成
- W11：内容营销 + SEO
- W12：复盘 + 写产品白皮书

## 📝 考核任务

毕业项目的"考核"是**真实世界的反馈**，硬指标：

- [ ] **T1**：完整 [Product Spec](https://github.com/asana/product-roadmaps) 文档，含目标用户 / 痛点 / 解决方案 / KPI
- [ ] **T2**：架构设计文档（含数据流图 / 时序图 / DB schema / 部署拓扑）
- [ ] **T3**：MVP 上线，公网可访问，HTTPS + 自定义域名
- [ ] **T4**：≥ 100 个真实用户注册，DAU ≥ 10
- [ ] **T5**：≥ 50% 用户次日留存
- [ ] **T6**：完整可观测性（前端埋点 + 后端 trace + LLM trace）
- [ ] **T7**：单元 + 集成 + E2E 测试覆盖率 ≥ 70%
- [ ] **T8**：CI/CD 全自动化，commit → 部署 < 5 分钟
- [ ] **T9**：在掘金 / 小红书 / Hacker News 发产品文，至少 100 阅读
- [ ] **T10**：GitHub README ≥ 100 star（开源项目）

- [N] 个用户、[N] DAU、[N] 月活

## 🚫 常见踩坑

1. **过度工程化**：MVP 不需要 K8s，docker-compose 就够
2. **追求技术新潮**：用稳定的，别用 alpha 框架
3. **忽视前期调研**：没用户问题，做出来没人用
4. **闭门造车**：每周公开进度，建立用户社区
5. **完美主义**：80 分发布，再迭代到 95 分
6. **忽视成本**：上线前先算清单次请求成本，否则爆掉
