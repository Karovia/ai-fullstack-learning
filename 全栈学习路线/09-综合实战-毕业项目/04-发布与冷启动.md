# 发布与冷启动

> "做出来不算，被人用过才算。"

90% 的独立产品是被自己的开发者埋葬的——不是技术差，是从来没让任何人知道。本章带你把 MVP 真正"扔出去"。

## 一、发布前 30 项检查清单

### 域名与基础
- [ ] 主域名 + HTTPS（Cloudflare）
- [ ] www / 裸域 自动 301
- [ ] 邮箱发件域 SPF / DKIM / DMARC
- [ ] favicon + apple-touch-icon
- [ ] 404 / 500 自定义页面

### SEO
- [ ] sitemap.xml 自动生成
- [ ] robots.txt
- [ ] 每页 title / description / og:image
- [ ] JSON-LD 结构化数据（产品 / FAQ）
- [ ] Google Search Console 提交

### 监控告警
- [ ] Sentry 错误捕获 + Slack 告警
- [ ] Better Stack uptime（5 分钟探测）
- [ ] Langfuse LLM 调用追踪
- [ ] PostHog 事件埋点（注册 / 激活 / 转化）
- [ ] 月度账单告警（超 200% 自动停服）

### 法务
- [ ] 隐私政策（参考 Termly 生成）
- [ ] 服务条款
- [ ] Cookie 同意（GDPR）
- [ ] 数据删除/导出入口
- [ ] 联系邮箱（必须真实有人回）

### 测试 & 质量
- [ ] 关键路径 e2e（Playwright）
- [ ] Lighthouse Performance ≥ 90
- [ ] Lighthouse Accessibility ≥ 90
- [ ] 移动端真机测过（iPhone + Android）

### 防滥用
- [ ] 限流（IP / 用户 / 全局三级）
- [ ] 验证码（hCaptcha）
- [ ] LLM 调用每用户日额度
- [ ] 文件上传大小/类型限制

### 数据安全
- [ ] DB 自动备份每日 + 异地
- [ ] 备份恢复演练 1 次
- [ ] 密钥都在 secrets manager（不在 .env 提交）

## 二、冷启动渠道（按 ROI 排序）

### Tier 1（必上）

#### ProductHunt
- 用户：海外开发者、PM、设计师
- 上首页 → 单日 1k-5k 访客
- 详细攻略见下文

#### Hacker News（Show HN）
- 用户：极客、工程师
- 上 front page → 单日 5k-50k 访客
- 标题党不行，要诚恳：`Show HN: AskMyDocs – I built a local-first PDF chat tool because Notion AI made me cry`

#### 小红书（中文）
- 用户：女性、学生、白领
- 视觉系，截图 + 短视频
- 笔记模板见下文

### Tier 2（必上其中 2 个）

#### V2EX / 即刻
- V2EX：技术圈，"分享创造"节点
- 即刻：年轻人，氛围轻松，发产品+故事

#### Twitter / X
- 长期建号 + 发布日 thread
- @ 大 V 不如做朋友

#### LinkedIn
- to B / 出海 必须
- 发"造产品故事"比硬广效果好

### Tier 3（看精力）

#### B 站 / YouTube
- 录 5 分钟 demo
- 标题要有"我做了 X，每月赚 Y"这类钩子

#### 知乎
- 长文写"我怎么做的 X"
- 评论区发产品

#### 行业社群
- 找 5 个高质量微信群/Discord
- 直接发不会被踢的群（先观察一周）

#### 朋友圈
- 不要刷屏
- 一篇用心写的故事，胜过 10 条硬广

## 三、ProductHunt 发布完整指南

### T-7 天：准备
- 注册账号 ≥ 30 天，有过点赞、评论
- 找一个 Hunter（粉丝 1000+ 的人）
- 准备素材：
  - Logo 240×240
  - Gallery 4-6 张（横版 1270×760）
  - 60s GIF / 短视频
  - Tagline ≤ 60 字符
  - Description ≤ 260 字符

### T-3 天：预热
- Upcoming page 上线，收集订阅
- 邮件给 mailing list："周X 我们上 PH"
- Twitter/LinkedIn 预告

### Launch Day（太平洋时间 0:01）
- 当天 0:01 上线（北京时间下午 3-4 点）
- 第一小时：自己 + 团队 + 5 个铁哥们 upvote + 评论
- 全天每 2 小时回评论一次
- Twitter / 即刻 / 微信群 同步发
- **不要用机器人刷**：被取消资格 = 半年禁赛

### Launch +1
- 不管几名，发 thanks 帖
- 拿到 #1-#5 → 立刻申请 Golden Kitty
- 把数据（访客 / 注册 / 收入）发出去

### 真实数据参考
- Top 1：~1500 upvotes，3-5k 访客，~500 注册
- Top 5：~600 upvotes，1-2k 访客
- 没上首页：~50 访客

## 四、Show HN 怎么写

```
Show HN: AskMyDocs – chat with your local PDFs without uploading them

Hi HN, I'm [name]. I built this because I had 80 PDFs for my thesis 
and Cmd+F was killing me.

Different from ChatPDF/Humata: 
- Local-first (PDFs never leave your machine in the free tier)
- Cross-doc search 
- Cite exact pages

Stack: Next.js, FastAPI, pgvector, Claude Sonnet.

Free tier: 50 questions/month. Paid $9/mo for unlimited.

Would love feedback, especially on accuracy with academic PDFs.
```

要诚恳，写"我为什么做"+"和别人不同在哪"+"求批评"。**不要营销腔**。

## 六、邀请码 / 早鸟策略

### 邀请码
- 制造稀缺感
- "每邀请 1 人多 30 天"
- 起到天然推荐裂变

### 早鸟优惠
- 前 100 名 lifetime $49（正常 $9/月）
- 锁定一批死忠
- 拿到 Stripe 第一笔 → 心理意义巨大

### A/B 价格测试
PostHog Feature Flag 跑：50% 用户 $9，50% 用户 $14。3 周后看转化 + 付费意愿。

## 七、用户反馈渠道

### 选 1 个主战场
- **Discord**（开发者类）
- **微信社群**（中文 to C）
- **邮件列表**（专业 / to B）
- **Crisp / Intercom**（产品内嵌入）

### 必备：24h 内必回
你的第一波用户是金子。哪怕回"我看到了，明天给你答复"，也比 3 天不回好。

## 八、第一波用户怎么留住

### 个人化 onboarding
注册后 1 小时内手发邮件：
> "嘿 X，我是 AskMyDocs 的 [你]。看到你刚注册，问下你想用它解决什么具体问题？我可以帮你跑通第一个流程。"

100 封手发邮件 = 比 1 万封自动化邮件值钱。

### 24h 黄金期
新用户 24h 不激活，70% 流失。
- 注册后立刻引导上传第一份文档
- 1h 后没上传 → 邮件提醒
- 24h 后没上传 → 短邮件 + 1v1 帮助

## 九、数据分析：三大核心指标

### DAU / WAU / MAU
- DAU/MAU > 20% 算高黏性

### 留存
- D1 / D7 / D30
- D1 留存 ≥ 40% 才算 onboarding 过关

### 转化漏斗
```
访客 → 注册 → 激活 → 留存 → 付费
1000   200   100    50      5
        ▲     ▲     ▲       ▲
        20%   50%   50%     10%
```

每一段都要找最低值优化。

## 十二、案例：发布日的真实一天

学员小 R：
- 0:01 PT（15:01 北京）PH 上线
- 第 1 小时：拉了 5 个朋友 upvote，自己发 thread
- 第 3 小时：评论区被一个用户挑刺"中文支持差"，他立刻 hotfix 上线
- 当日：#4 / 1.2k upvotes / 3.5k 访客 / 380 注册 / 18 付费 = $162 MRR
- 后续 1 周：又涨 200 注册（PH 长尾流量 + 小红书）

不是所有人都能上 #4，但**当天 hotfix 这件事让用户记住了你**。这才是冷启动核心：让人觉得"这个开发者在乎"。
