# 真实毕业项目案例库

> "你不是在凭空创造，你是在站在 30 个成功项目的肩膀上挑选基因。"

本章收录 30 个真实存在、有用户、有数据的 AI / Web 全栈项目。每个项目都是独立开发者或小团队从 0 → 1 做起来的。看完这份名单，你会发现：**毕业项目不是一个虚构的练习，而是你下一个产品的胚胎**。

每个项目卡片包含：定位、团队、技术栈、用户量、关键技术、启发、链接。

---

## 🤖 AI Agent 类（10 个）

### 1. Cursor

- **定位**：AI-first 的 IDE，把 VS Code fork 出来重做了交互层。
- **团队**：Anysphere（4 人起家，2026 年估值 90 亿美元）
- **技术栈**：Electron + VS Code fork + Claude / GPT API + 自研代码索引
- **用户量**：2026 年 ARR 约 5 亿美元，付费用户百万级
- **关键技术**：代码全仓库 embedding 索引、tab-完成上下文压缩、多文件 diff 应用
- **启发**：**fork 一个成熟产品 + 注入 AI 是最快路径**。不要重写 IDE 内核，把精力花在差异化体验上。
- **链接**：[cursor.sh](https://cursor.sh) · 闭源

### 2. v0.dev

- **定位**：一句话生成 React + shadcn/ui 组件代码。
- **团队**：Vercel 内部 7 人小组
- **技术栈**：Next.js + AI SDK + 自训 / 微调模型 + Sandpack 实时预览
- **关键技术**：受限输出（强制 JSX + Tailwind + shadcn）、组件级流式生成
- **启发**：**约束生成空间是质量的关键**。不让 AI "什么都生成"，只让它生成你能保证可用的格式。
- **链接**：[v0.dev](https://v0.dev) · 闭源

### 3. Perplexity

- **定位**：AI 搜索引擎。
- **团队**：Aravind Srinivas + 3 联创
- **技术栈**：自建搜索基础设施 + LLM Orchestration + Next.js
- **用户量**：2026 年 月活超 1 亿，估值 90 亿美元
- **关键技术**：RAG 实时引用、多步推理（Pro Search）、追问上下文管理
- **启发**：**重做一个老市场（Google 主导）的勇气**。即使巨头垄断，AI 给了重新定义产品的机会。
- **链接**：[perplexity.ai](https://perplexity.ai)

### 4. Lex.page

- **定位**：AI 写作编辑器，给作家用的 GitHub Copilot。
- **团队**：Nathan Baschez（Every 联合创始人）+ 2 人
- **技术栈**：Next.js + ProseMirror + GPT API
- **关键技术**：富文本编辑器中的内联补全、写作风格记忆
- **启发**：**深耕一个垂直人群**（专业作者）远胜泛化 AI 助手。
- **链接**：[lex.page](https://lex.page)

### 5. Cody (Sourcegraph)

- **定位**：企业级代码 AI 助手，主打代码库上下文。
- **团队**：Sourcegraph
- **技术栈**：Sourcegraph 索引基础设施 + Claude API
- **关键技术**：跨仓库代码图谱、企业 SSO、按代码库训练
- **启发**：**B 端市场要靠"上下文 + 合规"赢**，不是模型。
- **链接**：[sourcegraph.com/cody](https://sourcegraph.com/cody)

### 6. tldraw

- **定位**：极简白板 + AI（Make Real：画图变可运行 HTML）。
- **团队**：Steve Ruiz + 6 人小团队
- **技术栈**：React + 自研 canvas 引擎 + Claude
- **关键技术**：绘图截图 → 多模态 LLM → HTML 生成回投白板
- **启发**：**老产品 + AI = 病毒级 demo**。tldraw 已经存在多年，"Make Real" 是 1 周做出来的功能，全网爆款。
- **链接**：[tldraw.com](https://tldraw.com) · GitHub: [tldraw/tldraw](https://github.com/tldraw/tldraw)

### 7. Bolt.new

- **定位**：浏览器里的全栈 AI 开发环境（StackBlitz 团队）。
- **团队**：StackBlitz
- **技术栈**：WebContainers（浏览器跑 Node）+ Claude
- **用户量**：上线 6 个月内 ARR 4000 万美元
- **关键技术**：WebContainer 让 AI 在浏览器里跑 npm install
- **启发**：**多年技术积累（WebContainer 已 5 年）+ AI 时机 = 爆发**。
- **链接**：[bolt.new](https://bolt.new)

### 8. Lovable

- **定位**：欧洲版 Bolt，"Build apps by chatting"。
- **团队**：瑞典团队
- **技术栈**：Next.js + Claude + Supabase 集成
- **用户量**：上线 8 周 ARR 1700 万美元
- **关键技术**：内置部署 + 数据库 + 鉴权一条龙
- **启发**：**地理 / 集成差异化**——同一时代相同产品，针对欧洲企业 + 一键部署，照样能起。
- **链接**：[lovable.dev](https://lovable.dev)

### 9. Devin (Cognition)

- **定位**：自主 AI 软件工程师。
- **团队**：Cognition Labs
- **技术栈**：自研 agent 框架 + 沙箱浏览器 + 终端
- **关键技术**：长任务规划、自我纠错、人机协作 UI
- **启发**：**演示视频先于产品 = 估值起飞**——Devin 早期靠一段 demo 视频融到 20 亿估值。
- **链接**：[devin.ai](https://devin.ai)

### 10. Replit Agent

- **定位**：Replit 内置的 AI agent，从描述到部署一条龙。
- **团队**：Replit
- **技术栈**：Replit Cloud + Claude
- **关键技术**：与现有 IDE 深度集成、即时部署

## 共性总结：为什么成功 + 你能借鉴什么

1. **大多数都不是凭空创造**，而是 "老需求 × 新技术"（AI、本地优先、实时协作、开源）。
2. **小团队优势 = 速度 × 审美**——Linear、Heptabase、Plausible 都是几人到几十人。
3. **开源是双赢**——既是 marketing，也是信任，也是招聘漏斗。
4. **垂直人群 > 泛用户**——Lex（作者）、Cursor（开发者）、Cal（自由职业者）。
5. **Demo 视频驱动早期增长**——Devin、Bolt、Lovable 都靠一条 demo 起势。
6. **从已有平台扩展**比从零做容易——Replit Agent、Tldraw Make Real。

## 独立开发者最常踩的 10 个坑

1. **过度规划**：花 3 个月想 idea，1 周写代码，0 用户。**正确顺序：1 周想 → 4 周做 → 持续找用户。**
2. **追求完美 MVP**：上线后才有反馈，**丑陋能用 > 漂亮没人用**。
3. **不收钱**：免费用户不给反馈，付费用户才会骂你。**第一天就让能付费**。
4. **目标人群太宽**："所有内容创作者"是没人——选 "在 Substack 写每周 newsletter 的科技作者"。
5. **重技术轻分发**：不写博客 / 不发推 / 不进社群 = 没人知道你存在。
6. **抄第一名**：第二名很难活——做差异化（地理、价格、垂直、风格）。
7. **不和用户聊天**：每周打 5 个用户电话比写 100 行代码值钱。
8. **过早烧钱**：广告 / 投流不如 SEO + 内容 + 社区。
9. **一个人扛**：找搭档 / 加 indie hackers 群 / 找 mentor。
10. **半年没流水就放弃**：复利曲线前 6 个月平的，第 7 个月开始陡。**给自己 12 个月最低周期**。

## 怎么用这份案例库

1. **快速读**：今天浏览全部 30 个，标记 5 个让你心跳加速的。
2. **深读**：每周精读 1 个，看他们的：
   - 早期 Twitter / 博客（"我是怎么做到的"复盘）
   - GitHub commit 历史（看 v0.0.1 多简陋）
   - Hacker News 上的 Show HN 帖
3. **抄结构**：选 1 个作为你毕业项目的"骨架参考"，但**做差异化**。
