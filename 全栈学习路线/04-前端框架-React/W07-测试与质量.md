## 一、测试金字塔（先建立直觉）

```
          ▲
         /E2E\          10%   慢、贵、最真实
        /-----\
       /  集成  \       20%   中等
      /---------\
     /   单元    \      70%   快、便宜、定位准
    /-------------\
```

| 层级 | 工具 | 跑什么 | 速度 |
| --- | --- | --- | --- |
| 单元 | Vitest / Jest | 一个函数、一个组件 | <1ms |
| 集成 | Vitest + RTL | 多组件协作 + Mock API | 几十 ms |
| E2E | Playwright / Cypress | 真浏览器跑用户流程 | 几秒 |

**口诀**：能用单元解决，绝不写 E2E。

## 二、Vitest 入门（5 分钟）

```bash
pnpm add -D vitest @testing-library/react @testing-library/user-event @testing-library/jest-dom jsdom
```

`vitest.config.ts`：
```ts
import { defineConfig } from 'vitest/config'
import react from '@vitejs/plugin-react'
export default defineConfig({
  plugins: [react()],
  test: { environment: 'jsdom', globals: true, setupFiles: './src/test/setup.ts' }
})
```

`src/test/setup.ts`：
```ts
import '@testing-library/jest-dom/vitest'
```

第一个测试：

```ts
// src/utils/math.test.ts
import { describe, it, expect, beforeEach } from 'vitest'
import { add } from './math'

describe('add', () => {
  let base = 0
  beforeEach(() => { base = 10 })

  it('两个正数相加', () => {
    expect(add(base, 2)).toBe(12)
  })
  it('负数相加', () => {
    expect(add(-1, -2)).toBe(-3)
  })
})
```

跑：`pnpm vitest`（watch 模式，按 `q` 退出）。

### 常用 expect

| 写法 | 含义 |
| --- | --- |
| `toBe(x)` | === |
| `toEqual({a:1})` | 深比较 |
| `toContain('x')` | 数组/字符串包含 |
| `toBeTruthy()` / `toBeFalsy()` | 真假 |
| `toThrow()` | 抛异常 |
| `toHaveBeenCalledWith(1,2)` | mock 被这样调用过 |

## 三、React Testing Library 哲学

> **"测用户能感知到的行为，别测实现细节。"**

❌ 反例（脆，改实现就挂）：
```ts
expect(wrapper.state().count).toBe(1)
expect(wrapper.find('button').props().onClick).toBeDefined()
```

✅ 正例（用户视角）：
```ts
import { render, screen } from '@testing-library/react'
import userEvent from '@testing-library/user-event'

it('点击按钮 +1', async () => {
  const user = userEvent.setup()
  render(<Counter />)
  const btn = screen.getByRole('button', { name: /加一/ })
  await user.click(btn)
  expect(screen.getByText('当前: 1')).toBeInTheDocument()
})
```

### 查询优先级（按这个顺序选）

| 优先级 | API | 适用 |
| --- | --- | --- |
| 1 | `getByRole` | 几乎所有可交互元素（button/textbox/heading/link） |
| 2 | `getByLabelText` | 表单输入（连带测了无障碍） |
| 3 | `getByPlaceholderText` | 没 label 的输入框 |
| 4 | `getByText` | 静态文本 |
| 5 | `getByDisplayValue` | input 的当前值 |
| 6 | `getByAltText` / `getByTitle` | 图片 / title |
| 最后 | `getByTestId` | 实在没辙才用 `data-testid` |

### get / query / find 三套

| 前缀 | 找不到 | 异步 | 用法 |
| --- | --- | --- | --- |
| `getBy*` | 抛错 | 否 | 必定存在 |
| `queryBy*` | 返回 null | 否 | **断言不存在**（`expect(...).toBeNull()`）|
| `findBy*` | 抛错 | 是（默认等 1s） | 异步出现的元素 |

### 异步

```ts
import { waitFor } from '@testing-library/react'
await waitFor(() => expect(screen.getByText('成功')).toBeInTheDocument())
// 或者
expect(await screen.findByText('成功')).toBeInTheDocument()
```

## 四、Mock：vi.mock / vi.fn

```ts
import { vi } from 'vitest'

// mock 一个函数
const fn = vi.fn().mockReturnValue(42)
fn(1, 2)
expect(fn).toHaveBeenCalledTimes(1)
expect(fn).toHaveBeenCalledWith(1, 2)

// mock 整个模块
vi.mock('@/lib/api', () => ({
  fetchUser: vi.fn().mockResolvedValue({ id: 1, name: 'Alice' })
}))
```

**问题**：mock 接口太脆——后端字段一改，所有 mock 全要改。
**解法**：MSW。

## 五、MSW（Mock Service Worker）

在**网络层**拦截 fetch，比 mock 函数真实得多——你的代码不用改一行。

```bash
pnpm add -D msw
```

```ts
// src/test/handlers.ts
import { http, HttpResponse } from 'msw'
export const handlers = [
  http.get('/api/users', () => HttpResponse.json([{ id: 1, name: 'Alice' }])),
  http.post('/api/login', async ({ request }) => {
    const body = await request.json() as any
    if (body.password === 'wrong') return new HttpResponse(null, { status: 401 })
    return HttpResponse.json({ token: 'abc' })
  })
]

// src/test/setup.ts
import { setupServer } from 'msw/node'
import { handlers } from './handlers'
const server = setupServer(...handlers)
beforeAll(() => server.listen())
afterEach(() => server.resetHandlers())
afterAll(() => server.close())
```

测试组件：

```tsx
it('登录成功跳转', async () => {
  const user = userEvent.setup()
  render(<LoginPage />)
  await user.type(screen.getByLabelText(/邮箱/), 'a@b.com')
  await user.type(screen.getByLabelText(/密码/), '12345678')
  await user.click(screen.getByRole('button', { name: /登录/ }))
  expect(await screen.findByText(/欢迎/)).toBeInTheDocument()
})
```

## 六、Playwright E2E

```bash
pnpm create playwright@latest
```

```ts
// tests/login.spec.ts
import { test, expect } from '@playwright/test'

test('登录 → 看到 Dashboard', async ({ page }) => {
  await page.goto('/login')
  await page.getByLabel('邮箱').fill('a@b.com')
  await page.getByLabel('密码').fill('12345678')
  await page.getByRole('button', { name: '登录' }).click()
  await expect(page.getByRole('heading', { name: 'Dashboard' })).toBeVisible()
})
```

跑：`pnpm playwright test --ui`（带可视化调试器，香）。

**常用动作**：`page.goto / locator / fill / click / waitFor / screenshot`。
**录制**：`pnpm playwright codegen localhost:3000`，操作一遍自动生成代码。

## 常见错误

| 现象 | 原因 | 解法 |
| --- | --- | --- |
| `Unable to find element by role: button` | 元素是 `<div onClick>` 没 role | 改成 `<button>` 或加 `role="button"` |
| `act() warning` | 状态变更没等异步完成 | 用 `await user.click` / `findBy` / `waitFor` |
| 测试结果随机失败 | beforeEach 没清干净状态 | `vi.restoreAllMocks()` / `server.resetHandlers()` |
| 覆盖率不动 | `coverage` 没配 include | `vitest.config.ts` 里 `coverage: { include: ['src/**'] }` |
| RTL 找不到 Suspense 内容 | 异步还没解析 | 用 `findBy*` |
