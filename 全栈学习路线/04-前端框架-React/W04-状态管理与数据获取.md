# W04 状态管理与数据获取

## 0. 状态分类——分清楚才不混乱

| 类型 | 例子 | 工具 |
|---|---|---|
| **本地状态** | 弹窗开关、输入框 | `useState` / `useReducer` |
| **跨组件状态** | 当前用户、主题 | Context（少）/ **Zustand**（多） |
| **服务端状态** | 后端列表、详情 | **TanStack Query** |
| **URL 状态** | 当前页码、过滤参数 | React Router `useSearchParams` |
| **表单状态** | 输入值、校验错误 | **React Hook Form**（W05） |

类比：Zustand 是「**客户端的小金库**」，Query 是「**服务端的取款机**」——两个工具，分工不同，**别混用**。

🚨 **2026 反模式**：把后端数据塞 Redux/Zustand 里手动同步——**不要这么做**，让 Query 管。

## 1. Zustand 基础

```bash
pnpm add zustand
```

### 1.1 创建 store

```tsx
// src/store/counter.ts
import { create } from 'zustand'

interface CounterState {
  count: number
  inc: () => void
  dec: () => void
  reset: () => void
}

export const useCounter = create<CounterState>((set) => ({
  count: 0,
  inc: () => set(s => ({ count: s.count + 1 })),
  dec: () => set(s => ({ count: s.count - 1 })),
  reset: () => set({ count: 0 }),
}))
```

### 1.2 在组件里用

```tsx
function Counter() {
  const count = useCounter(s => s.count)
  const inc = useCounter(s => s.inc)
  return <button onClick={inc}>{count}</button>
}
```

类比：Zustand store 是「**全局 useState**」——任何组件都能 `useCounter(s => s.xxx)` 拿到。**不需要 Provider**。

### 1.3 selector 优化

```tsx
// ❌ 拿整个 state，任何字段变都重渲
const { count } = useCounter()

// ✅ 只订阅 count，inc 调用不会让组件重渲
const count = useCounter(s => s.count)
```

「**一次订一个字段**」是 Zustand 性能秘诀。

### 1.4 浅比较多字段

```tsx
import { useShallow } from 'zustand/react/shallow'

const { name, age } = useCounter(useShallow(s => ({ name: s.name, age: s.age })))
```

### 1.5 在组件外读写 store

```tsx
useCounter.getState().count       // 读
useCounter.setState({ count: 5 }) // 写
useCounter.subscribe(s => console.log(s.count))
```

## 2. Zustand 中间件

### 2.1 persist：自动持久化

```tsx
import { create } from 'zustand'
import { persist } from 'zustand/middleware'

export const useAuth = create<AuthState>()(
  persist(
    (set) => ({
      user: null,
      login: (u) => set({ user: u }),
      logout: () => set({ user: null }),
    }),
    { name: 'auth-storage' }   // localStorage key
  )
)
```

刷新页面 user 还在——告别手写 localStorage 同步。

### 2.2 immer：直接改不报错

```bash
pnpm add immer
```

```tsx
import { create } from 'zustand'
import { immer } from 'zustand/middleware/immer'

export const useTodos = create<TodoState>()(
  immer((set) => ({
    todos: [],
    add: (text) => set(state => {
      state.todos.push({ id: Date.now(), text, done: false })  // ✅ 像可变对象一样改
    }),
    toggle: (id) => set(state => {
      const t = state.todos.find(t => t.id === id)
      if (t) t.done = !t.done
    }),
  }))
)
```

类比：immer 是「**Git 暂存区**」——你看似在改原对象，它在背后给你算 diff 生成新对象。

### 2.3 devtools：Redux DevTools 调试

```tsx
import { devtools } from 'zustand/middleware'

create<S>()(devtools(immer(persist(...))))   // 多个中间件可以叠
```

打开 Chrome 的 Redux DevTools 面板能看到每次 action 时 state 变化。

### 2.4 异步 action

直接 async 即可：

```tsx
export const useUser = create<UserState>((set) => ({
  user: null,
  loading: false,
  fetchUser: async (id) => {
    set({ loading: true })
    const r = await fetch(`/api/users/${id}`)
    const data = await r.json()
    set({ user: data, loading: false })
  },
}))
```

但**异步获取数据建议交给 TanStack Query**，下面讲。

## 3. Jotai 简介（原子化）

```bash
pnpm add jotai
```

```tsx
import { atom, useAtom } from 'jotai'

const countAtom = atom(0)
const doubleAtom = atom((get) => get(countAtom) * 2)  // 派生 atom

function Counter() {
  const [count, setCount] = useAtom(countAtom)
  const [double] = useAtom(doubleAtom)
  return <button onClick={() => setCount(c => c + 1)}>{count} / {double}</button>
}
```

类比：Zustand 是「**一个大对象**」，Jotai 是「**一堆小颗粒（atom）**」，自动追踪依赖，订阅最小化。

**何时选**：
- 状态结构平铺、互相独立 → Jotai
- 多字段聚合操作多 → Zustand

入门用 Zustand 即可，Jotai 等需要时再学。

## 4. TanStack Query：服务端状态王者

```bash
pnpm add @tanstack/react-query
```

```tsx
// src/main.tsx
import { QueryClient, QueryClientProvider } from '@tanstack/react-query'
const qc = new QueryClient()

createRoot(document.getElementById('root')!).render(
  <QueryClientProvider client={qc}><App /></QueryClientProvider>
)
```

### 4.1 useQuery 基础

```tsx
// src/api/todos.ts
export const fetchTodos = async (): Promise<Todo[]> => {
  const r = await fetch('https://jsonplaceholder.typicode.com/todos?_limit=10')
  if (!r.ok) throw new Error('fetch failed')
  return r.json()
}

// src/components/TodoList.tsx
import { useQuery } from '@tanstack/react-query'
import { fetchTodos } from '../api/todos'

export function TodoList() {
  const { data, isLoading, error, refetch } = useQuery({
    queryKey: ['todos'],
    queryFn: fetchTodos,
  })

  if (isLoading) return <p>加载中…</p>
  if (error) return <p>出错: {error.message}</p>

  return (
    <>
      <button onClick={() => refetch()}>刷新</button>
      <ul>{data!.map(t => <li key={t.id}>{t.title}</li>)}</ul>
    </>
  )
}
```

类比：Query 是「**带缓存的 fetch**」——同样的 queryKey 全局只发一次请求，结果缓存起来，多个组件共享。

### 4.2 缓存与刷新策略

```tsx
useQuery({
  queryKey: ['todos'],
  queryFn: fetchTodos,
  staleTime: 60_000,        // 1 分钟内视为新鲜，不重新请求
  gcTime: 5 * 60_000,       // 5 分钟无人订阅就垃圾回收
  refetchOnWindowFocus: true,  // 窗口聚焦时重新请求（默认 true）
  refetchOnReconnect: true,    // 网络恢复时重新请求
  retry: 3,                    // 失败重试 3 次
})
```

口诀：「**fresh / stale**」「**active / inactive**」决定一切。
- fresh = staleTime 内的数据，不会重新请求
- stale = 过期数据（仍展示），触发条件就重请求
- active = 至少有一个组件订阅
- inactive = 无人订阅，到 gcTime 就丢

### 4.3 useMutation：写操作

```tsx
import { useMutation, useQueryClient } from '@tanstack/react-query'

const qc = useQueryClient()
const addTodo = useMutation({
  mutationFn: async (text: string) => {
    const r = await fetch('/api/todos', {
      method: 'POST', headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ text }),
    })
    return r.json()
  },
  onSuccess: () => {
    qc.invalidateQueries({ queryKey: ['todos'] })   // 让 todos 重新拉
  },
})

// 调用
addTodo.mutate('learn query')
```

### 4.4 乐观更新

「**先改 UI 假装成功了，失败再回滚**」。

```tsx
useMutation({
  mutationFn: toggleTodoApi,
  onMutate: async (id) => {
    await qc.cancelQueries({ queryKey: ['todos'] })
    const prev = qc.getQueryData<Todo[]>(['todos'])
    qc.setQueryData<Todo[]>(['todos'], (old) =>
      old?.map(t => t.id === id ? { ...t, done: !t.done } : t)
    )
    return { prev }   // 传给 onError
  },
  onError: (_err, _id, ctx) => {
    qc.setQueryData(['todos'], ctx?.prev)   // 回滚
  },
  onSettled: () => qc.invalidateQueries({ queryKey: ['todos'] }),
})
```

### 4.5 useInfiniteQuery：无限滚动

```tsx
const { data, fetchNextPage, hasNextPage, isFetchingNextPage } = useInfiniteQuery({
  queryKey: ['posts'],
  queryFn: ({ pageParam = 1 }) => fetch(`/api/posts?page=${pageParam}`).then(r => r.json()),
  initialPageParam: 1,
  getNextPageParam: (last, all) => last.hasMore ? all.length + 1 : undefined,
})

return (
  <>
    {data?.pages.flatMap(p => p.list).map(item => <Item key={item.id} {...item} />)}
    {hasNextPage && <button onClick={() => fetchNextPage()}>加载更多</button>}
  </>
)
```

## 5. Zustand vs Query 决策表

| 场景 | 用谁 |
|---|---|
| 当前用户、主题、语言 | Zustand |
| 一次性的 UI 状态（弹窗、面板） | useState |
| 后端列表 / 详情 / 搜索结果 | Query |
| 表单输入 | RHF（W05） |
| 加购物车（先本地后同步） | Zustand + Query.mutation |
| 编辑后端数据 | Query.mutation + invalidateQueries |

铁律：**"凡是后端能给的，永远走 Query"**。

## 6. 实战：用 Zustand + Query 重构 W1 Todo

用 jsonplaceholder 当假后端。

```tsx
// src/api/todos.ts
const API = 'https://jsonplaceholder.typicode.com'
export interface Todo { id: number; title: string; completed: boolean }

export const fetchTodos = (): Promise<Todo[]> =>
  fetch(`${API}/todos?_limit=8`).then(r => r.json())

export const toggleTodo = async (todo: Todo): Promise<Todo> => {
  const r = await fetch(`${API}/todos/${todo.id}`, {
    method: 'PATCH',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ completed: !todo.completed }),
  })
  return r.json()
}
```

```tsx
// src/store/filter.ts
import { create } from 'zustand'

type Filter = 'all' | 'done' | 'todo'
interface State { filter: Filter; setFilter: (f: Filter) => void }

export const useFilter = create<State>((set) => ({
  filter: 'all',
  setFilter: (filter) => set({ filter }),
}))
```

```tsx
// src/components/TodoApp.tsx
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query'
import { fetchTodos, toggleTodo, Todo } from '../api/todos'
import { useFilter } from '../store/filter'

export default function TodoApp() {
  const qc = useQueryClient()
  const { filter, setFilter } = useFilter()
  const { data, isLoading } = useQuery({ queryKey: ['todos'], queryFn: fetchTodos })

  const m = useMutation({
    mutationFn: toggleTodo,
    onMutate: async (todo) => {
      await qc.cancelQueries({ queryKey: ['todos'] })
      const prev = qc.getQueryData<Todo[]>(['todos'])
      qc.setQueryData<Todo[]>(['todos'], (old) =>
        old?.map(t => t.id === todo.id ? { ...t, completed: !t.completed } : t)
      )
      return { prev }
    },
    onError: (_e, _v, ctx) => qc.setQueryData(['todos'], ctx?.prev),
  })

  if (isLoading) return <p>加载中…</p>

  const list = (data ?? []).filter(t => {
    if (filter === 'done') return t.completed
    if (filter === 'todo') return !t.completed
    return true
  })

  return (
    <div style={{ maxWidth: 480, margin: '40px auto' }}>
      <h1>Todo (Zustand + Query)</h1>
      <div>
        {(['all', 'todo', 'done'] as const).map(f => (
          <button key={f} onClick={() => setFilter(f)}
                  style={{ fontWeight: filter === f ? 'bold' : 'normal' }}>
            {f}
          </button>
        ))}
      </div>
      <ul>
        {list.map(t => (
          <li key={t.id}>
            <input type="checkbox" checked={t.completed}
                   onChange={() => m.mutate(t)} />
            <span style={{ textDecoration: t.completed ? 'line-through' : 'none' }}>
              {t.title}
            </span>
          </li>
        ))}
      </ul>
    </div>
  )
}
```

注意分工：
- **Filter 是客户端状态** → Zustand
- **Todo 列表是服务端状态** → Query
- 切 filter 不会重新请求 todos（数据已缓存）
- 点 checkbox 立刻看到变化（乐观更新）
