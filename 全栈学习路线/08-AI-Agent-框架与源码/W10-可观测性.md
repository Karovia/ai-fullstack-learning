# W10 可观测性

## 0. 一句话开场

evaluation 告诉你「Agent 整体好不好」，observability 告诉你「**线上正在发生什么**」。前者是体检，后者是心电监护——一个都不能少。

## 1. 为什么是 Agent 生死线

类比：餐厅。

- 服务员说"今天客人都说菜难吃"——你只知道结果
- 厨房有摄像头，能看到每一道菜的火候、刀工——你能改进

Agent 跑 1000 次，10 个失败。没有 trace，你只知道"失败率 1%"。有 trace，你能看到失败的那 10 次到底在哪一步、为什么、上下文是什么——这是改进的起点。

## 2. 三大支柱

- **Traces**：一次完整请求的所有步骤（哪个 LLM 调用、哪个工具、什么参数、什么输出、用了多少 token）
- **Metrics**：聚合数据（QPS、延迟、错误率、token 消耗）
- **Logs**：补充上下文（异常栈、业务日志）

Agent 场景下，trace > metrics > logs（因为 Agent 步骤是树状的，普通日志读不出结构）。

## 3. LangSmith

LangChain 自家的，最丝滑（如果你用 LangChain/LangGraph）。

### 3.1 接入

```bash
pip install langsmith langchain
export LANGCHAIN_TRACING_V2=true
export LANGCHAIN_API_KEY=ls__xxx
export LANGCHAIN_PROJECT=my-agent
```

只需设环境变量，所有 LangChain / LangGraph 调用自动 trace，零代码改动。

### 3.2 自定义 trace

```python
from langsmith import traceable

@traceable(name="my_business_step")
def step(x):
    return x * 2
```

### 3.3 Datasets / Experiments

LangSmith 把"线上 trace"和"离线 eval"打通：

- 线上发现一条失败 trace
- 一键加进 dataset
- 重跑新版本看是否修复

### 3.4 价格（2026 中）

- Developer：免费 5k trace / 月
- Plus：$39 / 月起，含 10w trace
- Enterprise：自托管 / SOC2

**适用**：小团队、LangChain 重度用户、希望最少配置。

## 4. Langfuse（开源自托管）

近 2 年最火的开源 LLM observability，能自托管也有云版。

### 4.1 自托管（Docker Compose）

```bash
git clone https://github.com/langfuse/langfuse.git
cd langfuse
cp .env.example .env  # 改下密码 / NEXTAUTH_SECRET
docker compose up -d
# 浏览器打开 http://localhost:3000，注册第一个账号自动成为 owner
```

### 4.2 SDK 接入

```bash
pip install langfuse
```

```python
import os
from langfuse import Langfuse, observe

os.environ["LANGFUSE_HOST"] = "http://localhost:3000"
os.environ["LANGFUSE_PUBLIC_KEY"] = "pk-..."
os.environ["LANGFUSE_SECRET_KEY"] = "sk-..."

lf = Langfuse()

@observe()
def search(q):
    return run_search(q)

@observe()
def llm_call(messages):
    return openai.chat.completions.create(...)

@observe()
def agent(question):
    docs = search(question)
    return llm_call([{"role": "user", "content": question + str(docs)}])

agent("北京天气")
```

`@observe()` 自动嵌套 span，UI 里能看到树形 trace。

### 4.3 OpenAI / LangChain 直接集成

```python
from langfuse.openai import OpenAI  # drop-in
client = OpenAI()

# 或者 LangChain
from langfuse.langchain import CallbackHandler
chain.invoke({"q": "..."}, config={"callbacks": [CallbackHandler()]})
```

### 4.4 评分 / Dataset / Experiments

- `lf.score(trace_id=..., name="user_thumb", value=1)`：把用户反馈写回 trace
- `Dataset` + `run_experiment`：和 LangSmith 类似的离线 eval 工作流

### 4.5 优劣

- 优势：开源、能自托管（数据合规）、价格便宜、UI 不输 LangSmith
- 劣势：需要自己运维（一台 4G 内存机就行）

## 6. OpenTelemetry GenAI 规范

2025 年 OpenTelemetry 社区推出 GenAI Semantic Conventions：

- `gen_ai.system`：openai / anthropic / ...
- `gen_ai.request.model`：gpt-4o
- `gen_ai.usage.input_tokens` / `output_tokens`
- `gen_ai.tool.name`、`gen_ai.tool.call.id`

意义：**所有 observability 厂商都用同一套字段**，将来换厂商不用重写埋点。

实战建议：埋点尽量遵守这个规范（OpenInference 库已实现），将来 Phoenix / Langfuse / Datadog 都能直接读。

## 8. 自定义 trace（不用框架）

最朴素的写法（适合学习原理）：

```python
import uuid, time, json, contextvars
_current = contextvars.ContextVar("span", default=None)

class Span:
    def __init__(self, name, parent=None):
        self.id, self.name = str(uuid.uuid4()), name
        self.parent_id = parent.id if parent else None
        self.start = time.time(); self.end = None
        self.attrs = {}
        self.children = []
    def __enter__(self):
        parent = _current.get()
        if parent: parent.children.append(self)
        self.tok = _current.set(self)
        return self
    def __exit__(self, *exc):
        self.end = time.time()
        _current.reset(self.tok)
        if self.parent_id is None:
            print(json.dumps(serialize(self), indent=2, default=str))

def serialize(s):
    return {"id": s.id, "name": s.name, "ms": (s.end-s.start)*1000,
            "attrs": s.attrs, "children": [serialize(c) for c in s.children]}

# 用法
with Span("agent") as a:
    with Span("llm") as l:
        l.attrs["tokens"] = 1234
        ...
    with Span("tool.search") as t:
        ...
```

这就是 OpenTelemetry SDK 的迷你版。

## 9. 关键指标看板

至少这几张图：

1. **请求量**（按天 / 小时）
2. **延迟分布**（P50 / P95 / P99 折线）
3. **失败率分类**（堆叠柱状：工具错 / 推理错 / 超时 / 其他）
4. **Token 消耗**（按模型 + 按 endpoint）
5. **成本**（按用户 / 按 feature）
6. **用户反馈**（拇指上 / 拇指下比例）
7. **每会话步数分布**（直方图）

切片维度：用户 / 业务线 / 模型版本 / 客户端版本 / 地区。

## 10. 告警阈值

| 告警 | 阈值（示例） | 动作 |
| --- | --- | --- |
| 错误率 | 5 分钟 > 5% | 立即 Page |
| P95 延迟 | 持续 10 分钟 > 30s | 排查 |
| 成本 | 单日 > 预算 150% | 限流 + 通知 |
| 工具调用失败率 | 单工具 > 10% | 检查上游 API |
| 用户负反馈 | 单小时 > 20 条 | 暂停灰度 |

阈值要看历史 baseline，别拍脑袋。建议用「7 天移动平均 ± 3σ」。

## 11. 数据隐私 / PII 脱敏

trace 默认会把 prompt / response 全存下来，注意：

- **传输前脱敏**：手机号 / 身份证 / 邮箱用 regex 替换
- **字段级开关**：敏感 endpoint 关闭 input/output 记录
- **存储期限**：明确保留 30 / 90 / 365 天后删除
- **访问控制**：RBAC，只有运维 + 模型团队能看 raw trace
- **合规**：欧盟 GDPR、中国《个人信息保护法》、医疗 HIPAA

```python
import re
def redact(text):
    text = re.sub(r"\d{11}", "[PHONE]", text)
    text = re.sub(r"\d{17}[\dXx]", "[ID]", text)
    text = re.sub(r"[\w.+-]+@[\w-]+\.[\w.-]+", "[EMAIL]", text)
    return text
```

## 12. 实战：自托管 Langfuse + 100 任务报告

### 12.1 步骤

1. `docker compose up -d` 启动 Langfuse
2. 把 W7 的 CrewAI 投资研究小组接入 Langfuse callback
3. 准备 100 个真实股票 ticker
4. 跑 7 天，每天 ~15 个
5. 生成报告：

```python
from langfuse import Langfuse
lf = Langfuse()
traces = lf.fetch_traces(from_timestamp="2026-06-12", limit=200).data

import pandas as pd
df = pd.DataFrame([{
    "id": t.id, "name": t.name,
    "ms": (t.end_time - t.start_time).total_seconds()*1000 if t.end_time else None,
    "tokens": t.usage.total if t.usage else 0,
    "cost": float(t.total_cost or 0),
    "score": (t.scores[0].value if t.scores else None),
} for t in traces])

print(df.describe())
print("失败率:", (df.score < 0.5).mean())
print("成本/天:", df.groupby(df.id.str[:10])["cost"].sum())
```

### 12.2 报告模板

- 一周流量 / 成本 / 延迟趋势
- 失败 top10 trace ID + 一句根因
- 各 Agent 角色的 token 占比饼图
- 改进建议（系统 prompt / 工具 / 模型选型）
