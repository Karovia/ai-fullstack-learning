# W03 · 认证与授权：把"谁"和"能干嘛"分清楚

> 类比：进公司大楼。**认证**是门口刷工卡（你是张三吗？），**授权**是楼层门禁（张三能进财务部吗？）。把这两件事写在一处的代码，最后一定崩。

## 1. 认证 vs 授权

- **Authentication（authn）**：你是谁。判断身份。
- **Authorization（authz）**：你能干嘛。判断权限。

经典错误：在登录接口里直接判断 `is_admin`，把两件事缠在一起。正确做法是：登录只验证身份发凭证；后续每个请求由"鉴权依赖"独立判断权限。

## 2. 密码存储：不可逆 + 加盐 + 慢哈希

**永远不要明文存密码**。也不要 MD5/SHA1——它们快到一秒能爆破上亿次。要用专为密码设计的"慢哈希"：bcrypt / argon2 / scrypt。

```bash
uv add "passlib[bcrypt]" argon2-cffi
```

```python
from passlib.context import CryptContext

pwd = CryptContext(schemes=["argon2", "bcrypt"], deprecated="auto")

hashed = pwd.hash("mypassword")          # 每次结果不同（含随机盐）
ok     = pwd.verify("mypassword", hashed)

# 历史密码用 bcrypt，新用户用 argon2，自动迁移
if pwd.needs_update(hashed):
    hashed = pwd.hash("mypassword")
```

加盐由库内置。argon2 内置 memory-hard 抗 GPU 加速，2026 首选。

## 3. JWT：无状态凭证

JWT 三段式：`header.payload.signature`，每段 base64url 编码。

```
eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiIxMiIsImV4cCI6MTcxOTk5OTk5OX0.k-7Y...
```

- **header**：算法（HS256 对称 / RS256 非对称）
- **payload**：`sub`、`exp`、`iat`、自定义字段
- **signature**：前两段 + 密钥 经 HMAC 签出，**防篡改**，不防偷看

调试：[jwt.io](https://jwt.io) 在线解析。**Payload 不加密**，不要塞密码。

```bash
uv add "python-jose[cryptography]"
```

```python
from datetime import datetime, timedelta, timezone
from jose import jwt, JWTError

SECRET = "change-me"  # 用 secrets.token_urlsafe(32) 生成，存 .env
ALG    = "HS256"

def make_token(user_id: int, minutes: int = 30) -> str:
    now = datetime.now(timezone.utc)
    payload = {"sub": str(user_id), "iat": now, "exp": now + timedelta(minutes=minutes)}
    return jwt.encode(payload, SECRET, algorithm=ALG)

def parse_token(token: str) -> int:
    try:
        data = jwt.decode(token, SECRET, algorithms=[ALG])
        return int(data["sub"])
    except JWTError:
        raise ValueError("invalid token")
```

## 4. Access Token + Refresh Token

短的 access token（15-30 分钟）+ 长的 refresh token（7-30 天）。

- **Access Token**：放内存 / sessionStorage，泄露危害小
- **Refresh Token**：HttpOnly Cookie 或后端持久化，专用接口刷新

```python
# 登录返回俩 token
@router.post("/auth/login")
def login(form: LoginIn, db = Depends(get_session)):
    user = authenticate(db, form.email, form.password)
    return {
        "access_token":  make_token(user.id, minutes=15),
        "refresh_token": make_refresh(user.id, days=7),
        "token_type": "bearer",
    }

@router.post("/auth/refresh")
def refresh(body: RefreshIn, db = Depends(get_session)):
    user_id = parse_refresh(body.refresh_token)
    # 检查数据库里这个 refresh 没被吊销（rotate 策略：旧的失效 + 发新的）
    return {"access_token": make_token(user_id), "refresh_token": rotate_refresh(user_id)}
```

JWT 难"主动失效"是真痛点。常见方案：
1. 短过期（最实用）
2. 黑名单（Redis 存吊销的 jti，到期自动过期）
3. 用户级版本号 token_version，密码改了就 +1，老 token 全失效

## 5. FastAPI OAuth2PasswordBearer

```python
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm
from fastapi import Depends, HTTPException, status

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/login")

@router.post("/auth/login")
def login(form: OAuth2PasswordRequestForm = Depends(), db = Depends(get_session)):
    user = authenticate(db, form.username, form.password)
    return {"access_token": make_token(user.id), "token_type": "bearer"}

def get_current_user(token: str = Depends(oauth2_scheme), db = Depends(get_session)):
    try:
        uid = parse_token(token)
    except ValueError:
        raise HTTPException(401, "invalid token", {"WWW-Authenticate": "Bearer"})
    user = db.get(User, uid)
    if not user or not user.active:
        raise HTTPException(401)
    return user
```

`/docs` 右上角 Authorize 按钮自动出现，输入用户名密码就能调试受保护接口。

## 6. OAuth 2.0 四种流程

- **授权码（Authorization Code）+ PKCE**：第三方登录主流，前端跳到 GitHub / Google 授权后拿 code 换 token
- **客户端凭证（Client Credentials）**：服务到服务调用，没有用户
- **设备码（Device Code）**：电视、CLI 登录，输入网址 + 一次性码
- **密码模式（Password Grant）**：已废弃，只用于自家应用

## 7. GitHub OAuth 完整实战

1. https://github.com/settings/developers → New OAuth App
   - Authorization callback URL: `http://localhost:8000/auth/github/callback`
2. 拿到 `client_id` / `client_secret` 写进 `.env`
3. 后端两个端点：

```python
import secrets, httpx
from fastapi import APIRouter, Request
from fastapi.responses import RedirectResponse

router = APIRouter()

@router.get("/auth/github/login")
def github_login(request: Request):
    state = secrets.token_urlsafe(16)
    request.session["oauth_state"] = state    # 防 CSRF
    url = (
        "https://github.com/login/oauth/authorize"
        f"?client_id={CID}&scope=read:user user:email&state={state}"
    )
    return RedirectResponse(url)

@router.get("/auth/github/callback")
async def github_cb(code: str, state: str, request: Request, db = Depends(get_session)):
    if state != request.session.get("oauth_state"):
        raise HTTPException(400, "state mismatch")
    async with httpx.AsyncClient() as client:
        r = await client.post(
            "https://github.com/login/oauth/access_token",
            data={"client_id": CID, "client_secret": CSEC, "code": code},
            headers={"Accept": "application/json"},
        )
        gh_token = r.json()["access_token"]
        u = await client.get(
            "https://api.github.com/user",
            headers={"Authorization": f"Bearer {gh_token}"},
        )
        info = u.json()
    user = upsert_user_from_github(db, info)   # 没有就创建，有就更新
    return {"access_token": make_token(user.id)}
```

前端只要点一下 "Login with GitHub" 跳到 `/auth/github/login` 即可。

## 8. RBAC（Role-Based Access Control）

```python
class Role(str, Enum):
    USER = "user"
    EDITOR = "editor"
    ADMIN = "admin"

# Depends 工厂：声明需要的角色
def require_roles(*roles: Role):
    def dep(user = Depends(get_current_user)):
        if user.role not in roles:
            raise HTTPException(403)
        return user
    return dep

@router.delete("/articles/{id}")
def delete_article(id: int, _ = Depends(require_roles(Role.EDITOR, Role.ADMIN))):
    ...
```

更细粒度：基于"权限码"（permissions）而非角色，角色只是权限的集合。例：`articles:delete:any`、`articles:delete:own`。

## 9. 邮箱验证 / 找回密码

**统一思路**：发"一次性令牌 → 用户点链接 → 后端校验 → 完成动作"。

```python
import secrets

def issue_email_token(user_id: int, purpose: str, ttl_min: int = 30) -> str:
    payload = {"sub": str(user_id), "purpose": purpose,
               "exp": datetime.now(timezone.utc) + timedelta(minutes=ttl_min),
               "jti": secrets.token_urlsafe(8)}
    return jwt.encode(payload, SECRET, algorithm=ALG)

# 注册后
token = issue_email_token(user.id, "verify_email")
send_email(user.email, f"https://app.com/verify?token={token}")

# 校验端点
@router.get("/auth/verify")
def verify(token: str, db = Depends(get_session)):
    data = jwt.decode(token, SECRET, algorithms=[ALG])
    if data["purpose"] != "verify_email": raise HTTPException(400)
    user = db.get(User, int(data["sub"]))
    user.email_verified = True
    db.add(user); db.commit()
    return {"ok": True}
```

找回密码同理，purpose 改 `reset_password`，再加一次性 jti 黑名单防重放。

from fastapi.security import APIKeyHeader

api_key_header = APIKeyHeader(name="X-API-Key", auto_error=False)

def get_api_caller(key: str | None = Depends(api_key_header), db = Depends(get_session)):
