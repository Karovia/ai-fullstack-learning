# 第 05 章 · 后端开发 FastAPI + Node（8 周）

> Python 用 **FastAPI**（AI 时代主流），JS 用 **Hono**（替代 Express，更现代）。两条路同时走，能应对任何项目。

## 🎯 章节目标

- 设计并实现完整 RESTful API（OpenAPI 3.1 标准）
- 掌握认证：JWT / OAuth2 / Session
- 中间件、依赖注入、异常处理
- 文件上传 / 流式响应 / WebSocket / SSE
- 后台任务 + 定时任务（Celery / BullMQ）
- 限流 / 缓存 / 日志 / 监控
- 全链路测试 + CI/CD

## 📅 周计划

| 周 | 主题 | 产出 |
|---|------|------|
| W1 | HTTP/REST/RESTful 设计 + FastAPI 入门 | 第一个 API |
| W2 | FastAPI 进阶：依赖注入 / 中间件 / 异常 | 完整路由模块 |
| W3 | 认证与授权：JWT + OAuth2 + RBAC | 用户系统 |
| W4 | 高级特性：WebSocket / SSE / 文件上传 / 后台任务 | 实时通知 |
| W5 | Node.js + Hono 入门到进阶 | 同款 API |
| W6 | 高性能：限流 / 缓存 / 日志 / 监控 | 全套基础设施 |
| W7 | 测试：单元 + 集成 + E2E | 测试报告 |
| W8 | 部署：Docker + Nginx + CI/CD | 生产部署 |

## 📖 详细内容

### W1 - HTTP & FastAPI 入门
- HTTP 基础（方法/状态码/Header/Cookie）
- REST 设计原则 + RESTful 误区
- OpenAPI 3.1 规范
- FastAPI 路由 + Pydantic 模型
- 自动文档（Swagger UI / ReDoc）

### W2 - FastAPI 进阶
- Depends 依赖注入 + 嵌套依赖
- 中间件 + CORS
- 异常处理（HTTPException + 全局 handler）
- 后台任务（BackgroundTasks）
- APIRouter + 模块化项目结构（参考 [fastapi-best-practices](https://github.com/zhanymkanov/fastapi-best-practices)）

### W3 - 认证与授权
- 密码哈希（passlib + bcrypt）
- JWT（python-jose）+ Refresh Token 策略
- OAuth2 密码模式 / 授权码模式
- 第三方登录（GitHub OAuth2）
- RBAC（基于角色的访问控制）
- API Key + Rate Limit

### W4 - 高级特性
- WebSocket（FastAPI 内置）
- Server-Sent Events（流式响应，AI 必用）
- 文件上传（含大文件分片）
- 静态文件 + StaticFiles
- 任务队列：Celery + Redis 或 ARQ + Redis

### W5 - Node.js + Hono
- Node.js 基础（fs / stream / cluster / event）
- 现代框架对比：Express / Fastify / Hono / Elysia
- **Hono**（Cloudflare 推、零依赖、Edge 友好）
- TypeScript 类型化路由
- Drizzle ORM + Bun runtime（可选）

### W6 - 工程化基础设施
- 限流：Redis + sliding window
- 缓存：Redis + 缓存穿透/雪崩/击穿
- 日志：结构化 JSON + ELK / Loki
- 监控：Prometheus + Grafana
- APM：Sentry / OpenTelemetry

### W7 - 测试
- pytest + httpx.AsyncClient
- 测试金字塔（单元 70% / 集成 20% / E2E 10%）
- TestContainers（用真实 DB 测试）
- Mock 第三方 API
- 覆盖率 + 测试报告

### W8 - 部署
- Dockerfile 多阶段构建
- docker-compose（本地）
- Nginx 反向代理 + HTTPS（Let's Encrypt）
- GitHub Actions CI/CD
- 部署平台：Railway / Fly.io / Render（学习友好）+ AWS/GCP（进阶）

## 📝 考核任务

- [ ] **T1**：用 FastAPI 实现 CRUD（用户/文章/评论），通过 Postman 测试
- [ ] **T2**：实现 JWT 登录注册 + Refresh Token + 邮箱验证
- [ ] **T3**：实现 GitHub OAuth2 第三方登录

- [ ] **T7**：写测试，覆盖率 ≥ 80%

## 🚀 章节大项目：`Conduit` Real World 后端 + 实时聊天

### 项目 A：Conduit（标准化练习）
按 [RealWorld 规范](https://github.com/gothinkster/realworld) 实现 Conduit（Medium 简版）后端 API：
- 用户注册登录、关注
- 文章 CRUD、点赞、收藏
- 评论、标签
- 测试通过 [官方 Postman 集合](https://github.com/gothinkster/realworld/tree/main/api)

**用 FastAPI 实现一遍 + Hono 实现一遍**（对比同一规格的两种实现）。

### 项目 B：实时聊天后端
- WebSocket 房间 + 私聊
- Redis Pub/Sub 多机分发
- 离线消息持久化
- 在线用户列表
- 消息已读未读
- 文件/图片消息（MinIO 对象存储）
- 流式 AI 回复（SSE，为下一章预热）

### 验收标准
- [ ] Conduit 通过官方测试集合 100%
- [ ] FastAPI 与 Hono 两版本均可工作
- [ ] 聊天后端支持 100+ 并发连接
- [ ] 全部 Docker 化，docker-compose up 一键启动
- [ ] CI 跑全套测试 + 自动部署
- [ ] OpenAPI 文档完整，能用 Swagger UI 直接调试
