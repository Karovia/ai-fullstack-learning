# W07 · 测试：让代码改了也敢上线

> 类比：测试是房子的"承重墙图纸"。没图纸，每次想砸墙重装都得提心吊胆；有图纸，半天就敢翻新整面墙。

## 1. 为什么写测试

- 改代码不慌（重构敢动手）
- 文档化行为（看 test 比看注释快）
- 重构时有安全网
- CI 红绿一目了然

测试金字塔（自下而上越多）：
- **单元测试**：函数级，最多最快
- **集成测试**：组件协作（API + DB）
- **E2E**：模拟真实用户

## 2. pytest 基础进阶

```bash
uv add --dev pytest pytest-asyncio pytest-cov httpx faker factory-boy
```

`pyproject.toml`：

```toml
[tool.pytest.ini_options]
asyncio_mode = "auto"
testpaths = ["tests"]
addopts = "-ra --strict-markers --strict-config"
markers = [
  "slow: 慢测试",
  "integration: 集成测试",
]
```

跑：`uv run pytest -v`，跑某 mark：`-m "not slow"`，并行：`uv add --dev pytest-xdist` + `pytest -n auto`。

## 3. Fixture：测试的"依赖注入"

```python
# tests/conftest.py
import pytest
from sqlmodel import SQLModel, Session, create_engine
from sqlmodel.pool import StaticPool
from app.main import app
from app.core.db import get_session
from httpx import AsyncClient, ASGITransport

@pytest.fixture(scope="session")
def engine():
    eng = create_engine("sqlite://", connect_args={"check_same_thread": False}, poolclass=StaticPool)
    SQLModel.metadata.create_all(eng)
    return eng

@pytest.fixture
def db(engine):
    with Session(engine) as s:
        yield s
        s.rollback()

@pytest.fixture
async def client(db):
    app.dependency_overrides[get_session] = lambda: db
    async with AsyncClient(transport=ASGITransport(app=app), base_url="http://test") as c:
        yield c
    app.dependency_overrides.clear()
```

scope 四档：function（默认每个测试新建）/ class / module / session。共享 DB 连接用 session，事务回滚用 function。

## 4. 测 FastAPI（异步）

```python
# tests/test_articles.py
import pytest

@pytest.mark.asyncio
async def test_create_article(client, auth_token):
    res = await client.post(
        "/api/articles",
        headers={"Authorization": f"Bearer {auth_token}"},
        json={"title": "Hi", "body": "world"},
    )
    assert res.status_code == 201
    data = res.json()
    assert data["title"] == "Hi"
    assert "id" in data

@pytest.mark.asyncio
async def test_unauthenticated(client):
    res = await client.post("/api/articles", json={"title": "x", "body": "y"})
    assert res.status_code == 401
```

`auth_token` 也写成 fixture，先注册一个测试用户后签发：

```python
@pytest.fixture
async def auth_token(client):
    await client.post("/auth/register", json={"email": "t@t.com", "password": "Passw0rd!"})
    res = await client.post("/auth/login", data={"username": "t@t.com", "password": "Passw0rd!"})
    return res.json()["access_token"]
```

@respx.mock
async def test_github_login(client):

## 7. 工厂模式（factory_boy）

每个测试都手写 `User(email=..., password=...)` 太烦。

```python
import factory
from app.models import User

class UserFactory(factory.Factory):
    class Meta:
        model = User

    email = factory.Sequence(lambda n: f"user{n}@example.com")
    password_hash = factory.LazyFunction(lambda: pwd.hash("Passw0rd!"))
    is_admin = False

# 用法
def test_admin_required(db):
    user = UserFactory(is_admin=False)
    db.add(user); db.commit()
    ...
```

变体：`factory.SubFactory`（关联对象）、`factory.Faker("name")`（fake 数据）。

## 9. 单元 vs 集成 vs E2E

- **单元**：纯函数 / 无 IO。`def calc_price(items: list[Item]) -> Decimal`
- **集成**：起真实 DB / Redis / queue，走完一段链路（API → service → DB）
- **E2E**：端到端，完整运行环境 + 真实 HTTP 客户端

E2E 用 Playwright 调真实 API：

```bash
uv add --dev playwright
uv run playwright install chromium
```

```python
from playwright.async_api import async_playwright

async def test_e2e_login_then_create():
    async with async_playwright() as p:
        browser = await p.chromium.launch()
        api = await browser.new_context(base_url="http://localhost:8000")
        # 调真实正在跑的服务
        login = await api.request.post("/auth/login", form={"username": "t@t.com", "password": "Passw0rd!"})
        token = (await login.json())["access_token"]
        res = await api.request.post("/api/articles", headers={"Authorization": f"Bearer {token}"},
                                     data={"title": "E2E", "body": "..."})
        assert res.status == 201
```

## 11. TDD 流程演示

红 → 绿 → 重构。给"文章列表分页"加排序：

1. **红**：写一个失败的测试

```python
async def test_list_articles_sorted_by_created_desc(client, db):
    UserFactory(); a1 = ArticleFactory(title="old", created_at=...)
    a2 = ArticleFactory(title="new", created_at=...)
    db.commit()
    res = await client.get("/api/articles?sort=-created_at")
    assert [a["title"] for a in res.json()] == ["new", "old"]
```

跑 → 失败（功能没实现）

2. **绿**：用最小代码让它过

```python
@router.get("")
def list_articles(sort: str = "-created_at", db = Depends(get_session)):
    col = articles.created_at if sort.endswith("created_at") else articles.id
    order = col.desc() if sort.startswith("-") else col.asc()
    return db.exec(select(Article).order_by(order)).all()
```

3. **重构**：抽出 `parse_sort`，加白名单防注入。再跑测试，依然绿。

## 12. 变异测试（mutmut）

测试有"假阳性"——覆盖率高但其实没断言到位。**变异测试**：自动改你代码里的 `>` 变 `>=`、`+` 变 `-`，看测试还能不能挂。挂得越多，测试质量越高。

```bash
uv add --dev mutmut
uv run mutmut run --paths-to-mutate app
uv run mutmut results
```

进阶工具，量级合适后再上。
