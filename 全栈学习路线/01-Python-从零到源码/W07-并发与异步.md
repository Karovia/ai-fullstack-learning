# W07 · 并发与异步 🚦

> 本周目标：搞清"并发 vs 并行"、"线程 vs 进程 vs 协程"，能写出比同步快 30 倍的异步爬虫，理解为什么 Python 多线程做 CPU 密集任务**没用**。

---

## 1. 并发 vs 并行：不是一回事 ⚖️

**类比**：
- **并发（concurrency）**：一个厨师同时煮面、炒菜、烧汤，他在三口锅之间**来回切换**（一个时刻只动一锅）。
- **并行（parallelism）**：三个厨师**同时**各炒一锅菜。

| 概念 | 同时刻执行的任务数 | 关键词 | Python 工具 |
|------|------------------|-------|------------|
| 并发 Concurrency | 1 个（快速切换） | 协程 / 单核多线程 | asyncio / threading |
| 并行 Parallelism | 多个 | 多核 / 多机 | multiprocessing |

> 💡 **划重点**：并发是"看起来同时"，并行是"真的同时"。

---

## 2. GIL：Python 多线程的"紧箍咒" 🔒

**一句话**：CPython 解释器有一把全局锁（**G**lobal **I**nterpreter **L**ock），同一时刻只允许**一个线程**执行 Python 字节码。

**类比**：办公室只有一支唯一的笔（GIL），不管你雇 10 个会计（线程），只有拿到笔的人能写账。

| 任务类型 | 多线程能加速吗？ | 为什么 |
|---------|----------------|-------|
| **CPU 密集**（算数学） | ❌ 不能 | 锁住 GIL 自己跑，其他线程在等 |
| **IO 密集**（爬虫、读文件） | ✅ 能 | 等 IO 时会释放 GIL，其他线程趁机跑 |

> 💡 **结论**：CPU 密集用 `multiprocessing`（每个进程一把锁，互不影响）；IO 密集用 `threading` 或 `asyncio`。

## 3. threading：一行代码起一个线程 🧵

```python
import threading
import time

def worker(name):
    print(f"{name} 开始")
    time.sleep(1)              # 模拟 IO
    print(f"{name} 结束")

t1 = threading.Thread(target=worker, args=("A",))
t2 = threading.Thread(target=worker, args=("B",))
t1.start(); t2.start()         # 同时启动
t1.join();  t2.join()          # 等两个都跑完
print("主线程退出")
```

**运行结果**（顺序可能不同）：
```
A 开始
B 开始
（暂停 1 秒）
A 结束
B 结束
主线程退出
```

总耗时约 **1 秒**，而不是 2 秒（IO 等待时切换）。

### Lock：保护共享资源

```python
import threading

counter = 0
lock = threading.Lock()

def add():
    global counter
    for _ in range(100_000):
        with lock:                     # 进入临界区，其他线程要等
            counter += 1

threads = [threading.Thread(target=add) for _ in range(10)]
for t in threads: t.start()
for t in threads: t.join()
print(counter)                          # 1000000 ✅
```

不加锁的话，`counter += 1` 不是原子操作（读→加→写），多个线程会"踩脚"导致最终值小于 100 万。

### Queue：线程安全的传话筒

```python
import threading, queue, time

q = queue.Queue()

def producer():
    for i in range(5):
        q.put(i)
        print(f"生产 {i}")
        time.sleep(0.1)

def consumer():
    while True:
        item = q.get()
        if item is None: break          # 哨兵值，结束信号
        print(f"消费 {item}")
        q.task_done()

threading.Thread(target=consumer, daemon=True).start()
producer()
q.put(None)                             # 通知消费者结束
```

## 4. multiprocessing：绕过 GIL 的方法 🚀

`multiprocessing` 启动**真正的子进程**，每个进程有自己的解释器和 GIL。

```python
from multiprocessing import Pool
import time

def cpu_work(n):
    return sum(i*i for i in range(n))

if __name__ == "__main__":             # Windows/Mac 必须有这一行
    nums = [10_000_000] * 4
    t = time.time()
    with Pool(4) as p:                 # 4 个进程
        results = p.map(cpu_work, nums)
    print(time.time() - t)             # 比单进程快约 4 倍
```

> ⚠️ 进程间通信比线程贵得多（要序列化），不要给小任务开一堆进程。

## 5. concurrent.futures：高级 API（推荐） 🎯

`threading` / `multiprocessing` 太啰嗦，`concurrent.futures` 给了一个统一接口：

```python
from concurrent.futures import ThreadPoolExecutor, ProcessPoolExecutor

def fetch(url):
    import urllib.request
    return len(urllib.request.urlopen(url).read())

urls = ["https://www.example.com"] * 10

# IO 密集 → 线程池
with ThreadPoolExecutor(max_workers=5) as ex:
    sizes = list(ex.map(fetch, urls))
print(sizes)
```

只要把 `ThreadPoolExecutor` 换成 `ProcessPoolExecutor` 就能切到多进程，**接口一模一样**。

## 6. asyncio：Python 异步的银弹 🌟

**类比**：餐厅老板雇一个超能服务员（事件循环）。每次客人点完菜，他不站着等菜出锅，而是**立刻去招呼下一桌**。菜好了厨师按铃他再回来端。一个人服务 50 桌就是 asyncio。

### 关键概念

| 名称 | 作用 |
|------|------|
| **事件循环** event loop | 那个"超能服务员"，调度所有协程 |
| **协程** coroutine | 用 `async def` 定义的函数 |
| **await** | "我要等 IO，先暂停，让别人跑" |
| **Task** | 把协程包成"可调度的"对象 |

### 第一个异步程序

```python
import asyncio

async def hello(name, delay):
    print(f"{name} 开始")
    await asyncio.sleep(delay)         # 让出控制权给事件循环
    print(f"{name} 结束")

async def main():
    await asyncio.gather(              # 同时跑多个协程
        hello("A", 1),
        hello("B", 2),
        hello("C", 1),
    )

asyncio.run(main())
```

**运行结果**（总耗时约 2 秒）：
```
A 开始
B 开始
C 开始
A 结束
C 结束
B 结束
```

### async / sync 不能直接混！

```python
async def fetch():
    import time
    time.sleep(3)        # ❌ 错！这是同步阻塞，整个事件循环卡死 3 秒
    await asyncio.sleep(3)  # ✅ 对！这是异步等待
```

> 💡 **划重点**：在协程里调用同步阻塞函数（`time.sleep` / `requests.get` / 复杂计算），等于"把超能服务员摁在椅子上 3 秒"。要么换异步库，要么用 `loop.run_in_executor` 把它扔到线程池。

## 7. 异步 HTTP：aiohttp / httpx ⚡

```python
# pip install aiohttp
import asyncio, aiohttp, time

async def fetch(session, url):
    async with session.get(url) as r:
        return await r.text()

async def main():
    urls = [f"https://httpbin.org/delay/1" for _ in range(10)]
    async with aiohttp.ClientSession() as s:
        results = await asyncio.gather(*(fetch(s, u) for u in urls))
    print("拿到", len(results), "个响应")

t = time.time()
asyncio.run(main())
print("耗时", time.time() - t)         # 约 1 秒，10 个请求并发！
```

如果用 `requests` 同步抓，要 10 秒。

## 8. 实战：同步 vs 多线程 vs 异步爬豆瓣 Top 250 🥊

```python
import time, requests, asyncio, aiohttp
from concurrent.futures import ThreadPoolExecutor

URLS = [f"https://movie.douban.com/top250?start={i*25}" for i in range(10)]
HEADERS = {"User-Agent": "Mozilla/5.0"}

# 1. 同步
def sync_run():
    t = time.time()
    [requests.get(u, headers=HEADERS) for u in URLS]
    print("同步:", time.time() - t)

# 2. 多线程
def thread_run():
    t = time.time()
    with ThreadPoolExecutor(10) as ex:
        list(ex.map(lambda u: requests.get(u, headers=HEADERS), URLS))
    print("多线程:", time.time() - t)

# 3. 异步
async def async_run():
    t = time.time()
    async with aiohttp.ClientSession(headers=HEADERS) as s:
        await asyncio.gather(*(s.get(u) for u in URLS))
    print("异步:", time.time() - t)

sync_run()
thread_run()
asyncio.run(async_run())
```

**典型结果**：
```
同步:    8.3 秒
多线程:  1.2 秒
异步:    0.9 秒
```

> 💡 异步比多线程更快、更省内存（一个线程默认 8MB 栈，1000 个协程才几 MB）。

## 9. asyncio.Queue 生产者消费者 🏭

```python
import asyncio, random

async def producer(q: asyncio.Queue):
    for i in range(5):
        await asyncio.sleep(random.random())
        await q.put(i)
        print(f"生产 {i}")
    await q.put(None)                  # 哨兵：通知消费者结束

async def consumer(q: asyncio.Queue):
    while True:
        item = await q.get()
        if item is None: break
        await asyncio.sleep(0.5)
        print(f"消费 {item}")

async def main():
    q = asyncio.Queue()
    await asyncio.gather(producer(q), consumer(q))

asyncio.run(main())
```

### 🎯 动手练习 7.1
写一个异步爬虫：抓 https://httpbin.org/uuid 共 50 次，**最多 10 个并发**，计时。

<details>
<summary>👀 参考答案</summary>

```python
import asyncio, aiohttp, time

async def fetch(s, sem, url):
    async with sem:                    # 信号量控制并发数
        async with s.get(url) as r:
            return await r.json()

async def main():
    sem = asyncio.Semaphore(10)
    async with aiohttp.ClientSession() as s:
        urls = ["https://httpbin.org/uuid"] * 50
        results = await asyncio.gather(*(fetch(s, sem, u) for u in urls))
    print(len(results))

t = time.time(); asyncio.run(main()); print(time.time() - t)
```
**解析**：`Semaphore(10)` 像个 10 张通行证的口子，没拿到证就 await 等。

## 10. 定时任务：asyncio 周期调度 ⏰

```python
import asyncio, datetime

async def heartbeat():
    while True:
        print(datetime.datetime.now())
        await asyncio.sleep(1)         # 每 1 秒一次

async def main():
    task = asyncio.create_task(heartbeat())
    await asyncio.sleep(5)             # 跑 5 秒
    task.cancel()                       # 取消任务

asyncio.run(main())
```

## 🐛 常见错误集合

| 错误 | 原因 | 解决 |
|------|------|------|
| `RuntimeError: There is no current event loop` | 没用 `asyncio.run` | 用 `asyncio.run(main())` 启动 |
| 协程里阻塞 3 秒整体卡 3 秒 | 用了同步 IO | 换 aiohttp / `run_in_executor` |
| `coroutine was never awaited` | 调了 `f()` 没 `await` | 加 `await` 或 `asyncio.create_task` |
| 多线程 `counter` 不对 | 没加锁 | 用 `with lock:` 包起来 |
| `Pool` 必须放 `if __name__ == "__main__"` | Windows/Mac 子进程会重新 import | 加上这行守卫 |
