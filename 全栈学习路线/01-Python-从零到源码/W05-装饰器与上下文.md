# W05 · 装饰器与上下文管理器 🎁

> 这一周，你会读到很多写了 5 年 Python 的人都讲不清的概念 — **闭包、装饰器、上下文管理器**。
> 别紧张：它们只是**几个朴素的想法**叠在一起。一旦想通，你的代码就会变得"安静而强大"。

---

## 1. 函数是一等公民 🥇

**类比**：在 Python 里，函数就是个"普通对象" — **能装进变量、能当参数、能当返回值**，跟数字、字符串一样普通。

```python
def hello():
    return "hi"

f = hello             # 把函数本身赋给变量（不要写 ()）
print(f())            # hi

def call(fn):         # 把函数当参数
    return fn()
print(call(hello))    # hi

def factory():        # 把函数当返回值
    def inner():
        return "from inner"
    return inner

g = factory()
print(g())            # from inner
```

> 💡 这个能力是装饰器的基石。**记住**：不带 `()` 是"传函数本身"，带 `()` 是"调用并取结果"。

---

## 2. 闭包：函数把"出生时的环境"打包带走 📦

**类比**：闭包就像一个**带便当盒的人** — 走到哪儿都带着自己的环境（外层变量）。

```python
def make_counter():
    count = 0           # 外层变量
    def inner():
        nonlocal count  # 声明：要修改外层变量
        count += 1
        return count
    return inner

c = make_counter()
print(c(), c(), c())    # 1 2 3
```

`inner` 离开了 `make_counter`，但仍然能访问 `count` — 这就是**闭包**。

### 闭包的典型用例
```python
def make_adder(n):
    def add(x):
        return x + n
    return add

add5 = make_adder(5)
add10 = make_adder(10)
print(add5(3), add10(3))   # 8 13
```

> 💡 一句话：**闭包 = 函数 + 它所记住的外部变量**。

## 3. 装饰器：给函数"装个外壳" 🎀

**类比**：装饰器就是**手机壳** — 不改原手机，但加上保护、卡槽、支架等附加功能。

### 3.1 不用 `@` 的版本（先理解）
```python
def shout(fn):
    def wrapper():
        return fn().upper() + "!!!"
    return wrapper

def hello():
    return "hi"

hello = shout(hello)         # 用包装后的替换原函数
print(hello())               # HI!!!
```

### 3.2 用 `@` 的版本（语法糖）
```python
def shout(fn):
    def wrapper():
        return fn().upper() + "!!!"
    return wrapper

@shout                # 等价 hello = shout(hello)
def hello():
    return "hi"

print(hello())        # HI!!!
```

> 💡 `@xxx` 写在 `def f` 上一行，就是 `f = xxx(f)` 的简写。

## 4. 第一个实用装饰器：@timer ⏱️

```python
import time
from functools import wraps

def timer(fn):
    @wraps(fn)                       # 保留原函数的名字 / 文档
    def wrapper(*args, **kwargs):    # 通用：接任何参数
        start = time.perf_counter()
        result = fn(*args, **kwargs) # 调用原函数
        cost = time.perf_counter() - start
        print(f"{fn.__name__} 用时 {cost:.4f}s")
        return result
    return wrapper

@timer
def slow_sum(n):
    return sum(range(n))

print(slow_sum(1_000_000))
# slow_sum 用时 0.0123s
# 499999500000
```

> 💡 **`@wraps`** 极重要 — 不加的话，被装饰的函数会"丢名字"，调试时一脸懵。

## 5. 带参数的装饰器：再套一层 🎚️

需求：让 `@retry(times=3)` 自动重试 3 次。

```python
import time, random
from functools import wraps

def retry(times=3, delay=0.5):
    def decorator(fn):
        @wraps(fn)
        def wrapper(*args, **kwargs):
            for i in range(times):
                try:
                    return fn(*args, **kwargs)
                except Exception as e:
                    print(f"第 {i+1} 次失败：{e}")
                    time.sleep(delay)
            raise RuntimeError(f"{fn.__name__} 重试 {times} 次仍失败")
        return wrapper
    return decorator

@retry(times=3, delay=0.2)
def maybe_fail():
    if random.random() < 0.7:
        raise ValueError("network error")
    return "ok"

print(maybe_fail())
```

**结构**：装饰器**带参数 = 三层函数**。`retry(...)` 返回真正的装饰器 `decorator`。

## 6. 类装饰器与基于类的装饰器 🏗️

```python
from functools import wraps

class CountCalls:
    def __init__(self, fn):
        wraps(fn)(self)
        self.fn = fn
        self.n = 0
    def __call__(self, *args, **kwargs):
        self.n += 1
        print(f"{self.fn.__name__} 被调用第 {self.n} 次")
        return self.fn(*args, **kwargs)

@CountCalls
def greet(name):
    print(f"hi, {name}")

greet("a"); greet("b")
# greet 被调用第 1 次
# hi, a
# greet 被调用第 2 次
# hi, b
```

> 💡 类装饰器靠 `__call__` 让实例"像函数一样"被调用。

## 7. 内置好用的装饰器 🛠️

### `@lru_cache`：自动缓存
```python
from functools import lru_cache

@lru_cache(maxsize=None)
def fib(n):
    if n < 2: return n
    return fib(n-1) + fib(n-2)

print(fib(50))     # 12586269025  极快（没有 cache 会跑到天荒地老）
```

### `partial`：固定部分参数
```python
from functools import partial
def power(base, exp): return base ** exp

square = partial(power, exp=2)
cube = partial(power, exp=3)
print(square(5), cube(2))   # 25 8
```

## 8. 上下文管理器：自动清理的助手 🧹

**类比**：上下文管理器像**酒店退房系统** — 你"入住" → 用完就 "退房"，不会忘记关灯关门。

```python
with open("a.txt", "w") as f:
    f.write("hi")
# 离开 with 块，无论是否报错，f 都会被关闭
```

### 自定义：实现 `__enter__` / `__exit__`
```python
class Timer:
    def __enter__(self):
        import time
        self.start = time.perf_counter()
        return self                        # 这就是 as 后面的那个对象

    def __exit__(self, exc_type, exc_val, tb):
        import time
        self.cost = time.perf_counter() - self.start
        print(f"耗时 {self.cost:.4f}s")
        # return True  # 返回 True 会"吞掉"异常，谨慎使用

with Timer() as t:
    sum(range(10_000_000))
# 耗时 0.1234s
```

| 方法 | 作用 |
|------|------|
| `__enter__` | 进入 with 时执行，返回值绑定到 as |
| `__exit__` | 离开时执行，3 个参数收异常信息 |

## 9. `@contextmanager`：偷懒写上下文管理器 ⚡

```python
from contextlib import contextmanager
import time

@contextmanager
def timer(label="任务"):
    start = time.perf_counter()
    try:
        yield                       # ← yield 之前是 enter，之后是 exit
    finally:
        print(f"{label} 耗时 {time.perf_counter()-start:.4f}s")

with timer("求和"):
    sum(range(10_000_000))
```

> 💡 `yield` 把函数"切两段"。`with` 进入时跑 yield 之前，退出时跑 yield 之后。

## 10. 异常处理体系 🚨

```python
try:
    n = int(input("数字："))
    print(10 / n)
except ValueError:
    print("不是有效数字")
except ZeroDivisionError:
    print("除以零了")
except Exception as e:           # 兜底（不要省略变量 e）
    print("其他错误", e)
else:
    print("没异常才执行")
finally:
    print("无论如何都执行（清理）")
```

| 子句 | 何时执行 |
|------|---------|
| `try` | 主代码 |
| `except` | 出错时 |
| `else` | 没出错时 |
| `finally` | 永远执行（清理资源） |

### 自定义异常
```python
class InsufficientFundsError(Exception):
    """余额不足"""

def withdraw(balance, n):
    if n > balance:
        raise InsufficientFundsError(f"想取 {n}，只有 {balance}")
    return balance - n

try:
    withdraw(100, 200)
except InsufficientFundsError as e:
    print("被捕获：", e)
```

### 异常链（raise from）
```python
try:
    int("abc")
except ValueError as e:
    raise RuntimeError("解析配置失败") from e
# 报错信息会同时显示原始 ValueError 和 RuntimeError
```

> 💡 **不要 catch 所有异常然后 pass** — 这是工程灾难。要么处理，要么让它抛上去。

## 11. 实战 1：手写 5 个装饰器 ✋

```python
import time, json
from functools import wraps

# 1. timer
def timer(fn):
    @wraps(fn)
    def wrapper(*a, **kw):
        s = time.perf_counter()
        r = fn(*a, **kw)
        print(f"[timer] {fn.__name__} {time.perf_counter()-s:.4f}s")
        return r
    return wrapper

# 2. retry
def retry(times=3):
    def deco(fn):
        @wraps(fn)
        def wrapper(*a, **kw):
            for i in range(times):
                try: return fn(*a, **kw)
                except Exception as e:
                    print(f"第{i+1}次失败 {e}")
            raise
        return wrapper
    return deco

# 3. cache（用 dict 自己写一个）
def cache(fn):
    store = {}
    @wraps(fn)
    def wrapper(*a):
        key = json.dumps(a, default=str)
        if key not in store:
            store[key] = fn(*a)
        return store[key]
    return wrapper

# 4. deprecated
def deprecated(reason=""):
    def deco(fn):
        @wraps(fn)
        def wrapper(*a, **kw):
            print(f"[弃用] {fn.__name__} {reason}")
            return fn(*a, **kw)
        return wrapper
    return deco

# 5. validate（参数校验）
def validate(**rules):
    def deco(fn):
        @wraps(fn)
        def wrapper(*a, **kw):
            for k, t in rules.items():
                if k in kw and not isinstance(kw[k], t):
                    raise TypeError(f"{k} 必须是 {t.__name__}")
            return fn(*a, **kw)
        return wrapper
    return deco

@timer
@cache
def fib(n):
    if n < 2: return n
    return fib(n-1) + fib(n-2)

@validate(n=int)
def square(*, n):
    return n*n

print(fib(20))       # 第一次计算 + 缓存，再调即从 cache 读
print(square(n=5))   # 25
# print(square(n="x"))  # 抛 TypeError
```

## 12. 实战 2：文件操作上下文管理器 📂

```python
from contextlib import contextmanager
from pathlib import Path

@contextmanager
def safe_writer(path, encoding="utf-8"):
    """写入临时文件，成功才 rename — 防止半截文件覆盖原文件。"""
    path = Path(path)
    tmp = path.with_suffix(path.suffix + ".tmp")
    f = tmp.open("w", encoding=encoding)
    try:
        yield f
        f.close()
        tmp.replace(path)        # 原子替换
    except Exception:
        f.close()
        tmp.unlink(missing_ok=True)
        raise

with safe_writer("config.json") as f:
    f.write('{"debug": true}')
```

## 13. 实战 3：安全的资源释放 🔒

```python
from contextlib import contextmanager

class FakeDB:
    def __init__(self): self.connected = False
    def connect(self): self.connected = True; print("DB 连接")
    def close(self):  self.connected = False; print("DB 关闭")
    def query(self, sql): print("执行：", sql)

@contextmanager
def db_session():
    db = FakeDB()
    db.connect()
    try:
        yield db
    finally:
        db.close()

with db_session() as db:
    db.query("SELECT * FROM users")
    # raise RuntimeError("故意出错")   # 即使报错，DB 也会关
```

## 🐛 常见错误集合

| 错误 | 原因 |
|------|------|
| 装饰器丢函数名 | 没用 `@functools.wraps` |
| 闭包 `UnboundLocalError` | 修改外层变量没用 `nonlocal` |
| 带参数装饰器写错层数 | 记住"带参 = 三层函数" |
| `with` 没退出资源 | 自定义类忘写 `__exit__` |
| catch 了 Exception 然后 pass | 静默吞错，调试地狱 |
| `__exit__` return True | 会吞异常，慎用 |
