> 本周目标：彻底搞懂 `for x in something` 背后的"魔法"，能写出节省内存的生成器、能在 1GB 大文件上做处理而内存不爆炸，能熟练用 `itertools` 把一行代码顶十行 for 循环。

## 1. 一切从 `for` 循环说起 🚶

你已经写过无数次：

```python
for x in [1, 2, 3]:
    print(x)
```

但你有没有想过：Python **凭什么**知道要"一个一个"把元素拿出来？换成字符串、文件、字典也都能 for，背后到底有什么"统一接口"？

**类比**：你去自助餐厅取盘子。柜台口有个出盘机，按一下"咔哒"出一个，再按"咔哒"再出一个，出完之后亮红灯说"没了"。
- "出盘机" = **迭代器**（iterator）
- "整柜盘子" = **可迭代对象**（iterable）
- "按一下" = `next()`
- "亮红灯" = 抛出 `StopIteration`

---

## 2. iterable vs iterator：两个长得很像的家伙 🪞

| 概念 | 定义 | 例子 | 关键方法 |
|------|------|------|---------|
| **iterable**（可迭代对象） | 能"被 for"的东西 | list / dict / str / file | `__iter__()` 返回一个迭代器 |
| **iterator**（迭代器） | 真正"一个一个吐"的家伙 | `iter([1,2,3])` 的返回值 | `__next__()` 每次吐一个 |

```python
nums = [1, 2, 3]          # nums 是 iterable，但不是 iterator
it = iter(nums)            # 调用 iter() → 拿到一个迭代器
print(next(it))            # 1
print(next(it))            # 2
print(next(it))            # 3
print(next(it))            # 抛 StopIteration
```

**运行结果**：
```
1
2
3
Traceback (most recent call last):
  ...
StopIteration
```

> 💡 **划重点**：`for x in nums:` 等价于 `it = iter(nums); while True: try: x = next(it) except StopIteration: break`。Python 帮你藏起来了 try/except。

### 🎯 动手练习 6.1
用 `iter()` 和 `next()` 模拟 `for x in "abc": print(x)`，不要用 for。

<details>
<summary>👀 参考答案</summary>

```python

while True:
    try:
        x = next(it)
    except StopIteration:
        break
    print(x)
```
**解析**：所有的 for 循环底层就是这段代码，理解它你就理解了 Python 迭代协议。
</details>

## 3. 自己造一个迭代器 🛠️

迭代器协议只要求两件事：
1. 有 `__iter__` 方法，返回 `self`（迭代器自己）
2. 有 `__next__` 方法，每次返回下一个值，没了就 `raise StopIteration`

```python
class Countdown:
    """从 n 倒数到 1 的迭代器"""
    def __init__(self, n: int):
        self.n = n

    def __iter__(self):
        return self                       # 我自己就是迭代器

    def __next__(self):
        if self.n <= 0:
            raise StopIteration            # 没了，亮红灯
        cur = self.n
        self.n -= 1
        return cur

for x in Countdown(3):
    print(x)
```

**运行结果**：
```
3
2
1
```

### 🎯 动手练习 6.2
写一个 `MyRange(start, stop, step=1)`，模仿 `range`。

<details>
<summary>👀 参考答案</summary>

```python
class MyRange:
    def __init__(self, start, stop, step=1):
        self.cur, self.stop, self.step = start, stop, step

    def __iter__(self):
        return self

    def __next__(self):
        if (self.step > 0 and self.cur >= self.stop) or \
           (self.step < 0 and self.cur <= self.stop):
            raise StopIteration
        v = self.cur
        self.cur += self.step
        return v

print(list(MyRange(0, 10, 2)))   # [0, 2, 4, 6, 8]
```
</details>

## 4. 生成器函数：写迭代器的"傻瓜模式" 🎁

写迭代器要写 4 个方法，太累。Python 给了一个超级简单的语法糖：**`yield`**。

**类比**：`yield` 就像电视剧的"暂停键"。普通函数 `return` 一次就结束、内存释放；带 `yield` 的函数遇到 `yield` 会**暂停**，把值"递"出去，函数的局部变量、行号都被冻结；下次 `next()` 时**从那一行继续跑**。

```python
def countdown(n):
    while n > 0:
        yield n          # 把 n 递出去，函数暂停在这里
        n -= 1

for x in countdown(3):
    print(x)
```

**运行结果**：
```
3
2
1
```

> 💡 **划重点**：只要函数体里出现了 `yield`，这个函数就不再是"普通函数"，调用它**不会执行函数体**，而是返回一个**生成器对象**（一种迭代器）。函数体直到你 `next()` 才开始跑。

```python
g = countdown(3)
print(g)                # <generator object countdown at 0x...>
print(type(g))          # <class 'generator'>
print(next(g))          # 3
```

### 生成器函数 vs 普通函数

| 特性 | 普通函数 | 生成器函数 |
|------|---------|-----------|
| 关键字 | `return` | `yield` |
| 调用时执行函数体？ | ✅ 立即执行 | ❌ 返回 generator |
| 局部变量保留？ | ❌ 退出即销毁 | ✅ 暂停时冻结 |
| 内存占用 | 一次性算完 | 用一个算一个 |

## 5. 生成器表达式：一行写完一个生成器 ⚡

把列表推导的 `[]` 换成 `()` 就是生成器表达式：

```python
squares_list = [x*x for x in range(1_000_000)]   # 立刻算 1M 个数，吃几十 MB 内存
squares_gen  = (x*x for x in range(1_000_000))   # 一个都没算，几乎不占内存

print(sum(squares_gen))   # 一边吐一边加，内存恒定
```

> 💡 **划重点**：当你只需要"一遍 for"做求和、最大值、连接，**永远优先用生成器表达式**。

### 🎯 动手练习 6.3
读一个 1GB 的 CSV，统计第 3 列数字之和，**内存不能超过 100MB**。

<details>
<summary>👀 参考答案</summary>

```python
def col3_nums(path):
    with open(path, encoding="utf-8") as f:
        next(f)                          # 跳过表头
        for line in f:                   # 文件对象本身就是迭代器，一行一行读
            yield float(line.split(",")[2])

print(sum(col3_nums("big.csv")))         # 整个过程只占一行的内存
```
**解析**：`open()` 返回的文件对象就是一个迭代器，配合生成器函数，你可以处理任意大的文件。
</details>

## 6. yield from：委托给"子生成器" 🪜

```python
def chain(*iterables):
    for it in iterables:
        for x in it:
            yield x

# 等价写法（更短、更快、转发更彻底）
def chain2(*iterables):
    for it in iterables:
        yield from it

print(list(chain([1,2], (3,4), "ab")))    # [1, 2, 3, 4, 'a', 'b']
```

`yield from it` 不仅省一层 for，还会**自动转发** `send/throw/close`，写复杂协程时是必需的。

## 7. 协程基础：`send()` / `throw()` / `close()` 📡

生成器除了能"吐"，还能"接"东西：

```python
def echo():
    while True:
        msg = yield                    # yield 表达式：右边是吐出的值，"接"的是 send 进来的
        print(f"收到：{msg}")

g = echo()
next(g)                                # 必须先"启动"到第一个 yield
g.send("hi")
g.send("hello")
g.close()                              # 关闭生成器
```

**运行结果**：
```
收到：hi
收到：hello
```

> 💡 这就是 **协程** 的雏形。Python 后来用 `async/await` 替代了它（W07 详解），但理解 send/throw/close 对读老代码、读 asyncio 源码都很有帮助。

## 8. itertools：标准库的瑞士军刀 🔧

`itertools` 全是迭代器工具，**一个比一个好用**。

| 函数 | 作用 | 例子 | 输出 |
|------|------|------|------|
| `chain(a, b, ...)` | 串联多个可迭代 | `chain([1,2],[3,4])` | `1 2 3 4` |
| `islice(it, start, stop, step)` | 切片迭代器 | `islice(count(), 5, 10)` | `5 6 7 8 9` |
| `groupby(it, key)` | 按 key 分组（**先排序！**） | `groupby("aaabbc")` | `('a', aaa) ('b', bb) ('c', c)` |
| `product(a, b)` | 笛卡尔积 | `product("AB", [1,2])` | `A1 A2 B1 B2` |
| `combinations(it, r)` | 组合 | `combinations("ABC", 2)` | `AB AC BC` |
| `permutations(it, r)` | 排列 | `permutations("AB", 2)` | `AB BA` |
| `accumulate(it, func)` | 累计 | `accumulate([1,2,3,4])` | `1 3 6 10` |
| `cycle(it)` | 无限循环 | `cycle("AB")` | `A B A B …` |
| `takewhile(pred, it)` | 满足条件就取 | `takewhile(<5, count())` | `0 1 2 3 4` |
| `dropwhile(pred, it)` | 满足条件就丢 | `dropwhile(<3, [1,2,3,4,1])` | `3 4 1` |

```python
from itertools import chain, islice, groupby, product, accumulate

print(list(chain([1, 2], [3, 4])))                 # [1, 2, 3, 4]
print(list(islice(range(100), 5)))                 # [0, 1, 2, 3, 4]
print(list(accumulate([1, 2, 3, 4])))              # [1, 3, 6, 10]
print(list(product("AB", [1, 2])))                 # [('A',1), ('A',2), ('B',1), ('B',2)]

words = ["apple", "ant", "bee", "bat", "cat"]
words.sort(key=lambda w: w[0])                     # groupby 前必须排序！
for k, group in groupby(words, key=lambda w: w[0]):
    print(k, list(group))
```

**运行结果**：
```
[1, 2, 3, 4]
[0, 1, 2, 3, 4]
[1, 3, 6, 10]
[('A', 1), ('A', 2), ('B', 1), ('B', 2)]
a ['apple', 'ant']
b ['bee', 'bat']
c ['cat']
```

## 9. 实战：手写 itertools 核心函数 🏗️

```python
def my_range(start, stop=None, step=1):
    if stop is None:
        start, stop = 0, start
    cur = start
    while (step > 0 and cur < stop) or (step < 0 and cur > stop):
        yield cur
        cur += step

def my_enumerate(iterable, start=0):
    i = start
    for x in iterable:
        yield i, x
        i += 1

def my_zip(*iterables):
    iters = [iter(it) for it in iterables]
    while True:
        result = []
        for it in iters:
            try:
                result.append(next(it))
            except StopIteration:
                return
        yield tuple(result)

def my_chain(*iterables):
    for it in iterables:
        yield from it

# 验证
print(list(my_range(5)))                              # [0, 1, 2, 3, 4]
print(list(my_enumerate("abc", 1)))                   # [(1,'a'),(2,'b'),(3,'c')]
print(list(my_zip([1,2,3], "abc", [10,20,30])))       # [(1,'a',10),(2,'b',20),(3,'c',30)]
print(list(my_chain([1], (2,3), "ab")))               # [1, 2, 3, 'a', 'b']
```

## 10. 实战：1GB 大文件按行去重 🐘

```python
def unique_lines(path):
    seen = set()
    with open(path, encoding="utf-8") as f:
        for line in f:                  # f 是迭代器，一行一行来
            line = line.rstrip("\n")
            if line not in seen:
                seen.add(line)
                yield line

# 用法（边读边写，全程内存恒定）
with open("dedup.txt", "w", encoding="utf-8") as out:
    for line in unique_lines("big.log"):
        out.write(line + "\n")
```

> 💡 如果连 `seen` 集合都装不下（行特别多），可以换成 [Bloom Filter](https://en.wikipedia.org/wiki/Bloom_filter) 进一步省内存（W11 之后再聊）。

## 🐛 常见错误集合

| 错误 | 原因 | 解决 |
|------|------|------|
| `TypeError: 'generator' object is not subscriptable` | 生成器不能 `g[0]` | 先 `list(g)` 再下标 |
| 生成器只能 for 一次，第二次空 | 它**消费完就没了** | 重新调用生成器函数 |
| `StopIteration` 提前抛 | `__next__` 写错条件 | 仔细检查边界 |
| `groupby` 结果不对 | 没有先排序 | `sorted(it, key=...)` 后再 groupby |
| `next(g)` 直接报错 | 生成器还没启动 | 先 `next(g)` 启动到第一个 yield |
