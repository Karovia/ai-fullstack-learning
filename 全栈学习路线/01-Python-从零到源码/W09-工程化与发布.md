# W09 · 工程化与发布 📦

> 本周目标：把"能跑的脚本"升级为"能让别人 `pip install` 的真正项目"。配齐项目结构 / pyproject.toml / 单元测试 / CI / 发布到 PyPI。

---

## 1. 为什么要工程化？🤔

**类比**：W3 写的 `todo.py` 像一份**手写菜谱**——你自己用没问题，给别人就一脸懵。
工程化 = 把菜谱印成**菜单 + 配料表 + 烹饪流程**，任何人按照流程都能复刻。

| 没工程化 | 工程化 |
|---------|-------|
| 一个 .py 文件 | 完整目录 + pyproject.toml |
| 没有依赖清单 | 锁文件 uv.lock |
| 没有测试 | pytest + 覆盖率 |
| 没有 CI | GitHub Actions 自动测试 |
| 不能 pip install | 可发布到 PyPI |

## 2. 项目结构：src layout vs flat layout 🏗️

```
# flat layout（小项目，简单）
mytodo/
├── pyproject.toml
├── README.md
├── mytodo.py
└── tests/

# src layout（推荐，大项目）
mytodo/
├── pyproject.toml
├── README.md
├── LICENSE
├── src/
│   └── mytodo/
│       ├── __init__.py
│       ├── core.py
│       └── cli.py
└── tests/
    └── test_core.py
```

> 💡 **划重点**：用 src layout 的好处是——你**绝不会**意外 import 到当前目录的代码，必须 `pip install -e .` 后才能 import，**强制你像"用户"一样用自己的包**。

## 3. uv：现代 Python 工具链 ⚡

uv 由 ruff 同一团队（Astral）出品，用 Rust 写，比 pip / poetry 快 10-100 倍。**一个工具同时是 pip / venv / pip-tools / poetry / pyenv**。

```bash
# 安装 uv（macOS / Linux）
curl -LsSf https://astral.sh/uv/install.sh | sh

# 新建项目
uv init mytodo
cd mytodo

# 创建虚拟环境
uv venv

# 加依赖（自动改 pyproject.toml + 装包）
uv add pydantic
uv add --dev pytest ruff mypy

# 安装所有依赖（按 uv.lock 锁版本）
uv sync

# 跑命令（自动用项目的 venv）
uv run python -m mytodo
uv run pytest
```

## 4. pyproject.toml 全解 📜

```toml
# pyproject.toml
[project]
name = "mytodo"                            # PyPI 上的名字（不能重）
version = "0.1.0"                          # SemVer：主.次.补
description = "A tiny CLI todo built with Pydantic"
readme = "README.md"
requires-python = ">=3.10"
license = {text = "MIT"}
authors = [{name = "Your Name", email = "you@example.com"}]
keywords = ["todo", "cli", "pydantic"]
classifiers = [
    "Programming Language :: Python :: 3",
    "License :: OSI Approved :: MIT License",
]
dependencies = [
    "pydantic>=2.0",
    "click>=8.0",
]

[project.optional-dependencies]
dev = ["pytest", "ruff", "mypy"]

[project.scripts]
mytodo = "mytodo.cli:main"                 # 命令行入口

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[tool.ruff]
line-length = 100

[tool.pytest.ini_options]
testpaths = ["tests"]
```

> 💡 **`project.scripts`** 是发布后用户能直接在终端敲 `mytodo` 而不是 `python -m mytodo` 的关键。

## 5. ruff：项目规范的守门员 🛡️

```bash
uv add --dev ruff
uv run ruff check .          # 检查
uv run ruff check . --fix    # 自动修
uv run ruff format .         # 格式化（替代 black）
```

`pyproject.toml` 关键配置：

```toml
[tool.ruff.lint]
select = ["E", "F", "I", "UP", "B", "SIM", "N"]
ignore = ["E501"]            # 不限制行长（line-length 已配）
```

| 规则代码 | 含义 |
|---------|------|
| E | pycodestyle 风格错误 |
| F | pyflakes 未使用变量 |
| I | isort import 排序 |
| UP | pyupgrade 升级老语法 |
| B | flake8-bugbear 易错点 |
| SIM | 简化建议 |
| N | 命名规范 |

## 6. pytest：写测试不是负担 🧪

```python
# tests/test_core.py
import pytest
from mytodo.core import TaskList

def test_add_task():
    tl = TaskList()
    t = tl.add("写代码")
    assert t.id == 1
    assert t.title == "写代码"

def test_done_invalid_id():
    tl = TaskList()
    with pytest.raises(ValueError, match="任务不存在"):
        tl.done(999)

@pytest.mark.parametrize("title,ok", [
    ("hello", True),
    ("", False),
    ("x" * 200, False),
])
def test_title_validation(title, ok):
    tl = TaskList()
    if ok:
        tl.add(title)
    else:
        with pytest.raises(Exception):
            tl.add(title)
```

跑：

```bash
uv run pytest -v
uv run pytest --cov=mytodo --cov-report=term-missing   # 覆盖率
```

### fixture：可重用的测试数据

```python
@pytest.fixture
def sample_list():
    tl = TaskList()
    tl.add("a"); tl.add("b"); tl.add("c")
    return tl

def test_count(sample_list):
    assert len(sample_list.tasks) == 3
```

### 测试异步代码

```bash
uv add --dev pytest-asyncio
```

```python
import pytest

@pytest.mark.asyncio
async def test_async_fetch():
    result = await fetch("https://example.com")
    assert result.status == 200
```

## 7. pre-commit：提交前自动检查 🚧

```bash
uv add --dev pre-commit
```

`.pre-commit-config.yaml`：

```yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.5.0
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.6.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer
```

```bash
uv run pre-commit install         # 装 git hook
uv run pre-commit run --all-files # 全量跑一遍
```

之后每次 `git commit`，自动跑 ruff、自动修空白。

## 8. GitHub Actions CI：每次推送自动测试 🤖

`.github/workflows/test.yml`：

```yaml
name: test
on:
  push:
    branches: [main]
  pull_request:

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python: ["3.10", "3.11", "3.12"]
    steps:
      - uses: actions/checkout@v4
      - name: Install uv
        uses: astral-sh/setup-uv@v3
      - name: Setup Python ${{ matrix.python }}
        run: uv python install ${{ matrix.python }}
      - name: Install deps
        run: uv sync --all-extras
      - name: Lint
        run: uv run ruff check .
      - name: Type check
        run: uv run mypy src
      - name: Test
        run: uv run pytest --cov=mytodo
```

每次 `git push` 或开 PR，GitHub 会自动在 3 个 Python 版本下跑全套检查，绿勾才能合并。

## 9. SemVer：版本号怎么打 🔢

`MAJOR.MINOR.PATCH` —— 例如 `1.4.2`。

| 改动类型 | 升哪一位 | 例子 |
|---------|---------|------|
| **不兼容**改动（删了 API、改了行为） | MAJOR | 1.x → 2.0.0 |
| 新增功能（向后兼容） | MINOR | 1.4 → 1.5.0 |
| Bug 修复（向后兼容） | PATCH | 1.4.2 → 1.4.3 |

> 💡 0.x.y 是"试验期"，可以随便改。一旦发 1.0.0，就要遵守上面的规则。

## 10. 实战：发布 mytodo 到 TestPyPI 🚀

[TestPyPI](https://test.pypi.org/) 是 PyPI 的"练习场"，正式发布前先在这里跑一遍。

### 第一步：注册 + 拿 token
- 在 https://test.pypi.org 注册账号
- 设置里创建 API Token，复制 `pypi-...` 字符串
- 在 `~/.pypirc` 写：
```ini
[testpypi]
username = __token__
password = pypi-AgENdGVzdC5weXBpLm9yZ...
```

### 第二步：装打包工具
```bash
uv add --dev build twine
```

### 第三步：构建
```bash
uv run python -m build
# 产物在 dist/：
# mytodo-0.1.0.tar.gz       源码包
# mytodo-0.1.0-py3-none-any.whl  wheel 包
```

### 第四步：上传
```bash
uv run twine upload --repository testpypi dist/*
```

### 第五步：别人安装

```bash
pip install -i https://test.pypi.org/simple/ mytodo
mytodo --help                # 你的命令行工具能跑了！
```

### 正式发布到 PyPI

把 `--repository testpypi` 去掉、token 换成 https://pypi.org 的，重复第四步。

## 🐛 常见错误集合

| 错误 | 原因 | 解决 |
|------|------|------|
| `ModuleNotFoundError: mytodo` | 没装包 | `uv pip install -e .` |
| TestPyPI 上传 `400 already exists` | 同版本号重复上传 | 改一下版本号 |
| `pytest` 找不到测试 | 没在 pyproject 配 testpaths | 加 `testpaths = ["tests"]` |
| `import` 在 src layout 报错 | 没装可编辑 | `uv pip install -e .` |
| GitHub Actions 跑很久 | 没缓存 | 用 `astral-sh/setup-uv@v3` 自带缓存 |
