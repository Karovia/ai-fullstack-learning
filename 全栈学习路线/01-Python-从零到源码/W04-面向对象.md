# W04 · Python 面向对象 🐱🐶🐭

> 函数让你"复用一段逻辑"，而**类**让你"复用一个**有状态的概念**"。
> 这一周你会从"操作数据"，升级成"操作**对象**"。

---

## 1. 为什么要 OOP？🤔

**类比**：
- **过程式编程**：你像厨子，挨个处理食材。
- **OOP**：你管理一群"角色" — 每个角色（对象）有自己的**属性**（性格、技能）和**行为**（方法）。

想象你要写一个宠物乐园：
- 没有 OOP：每只宠物的属性散落在一堆变量里，方法也各自为政。
- 用 OOP：定义一个 `Pet` 类，每只小猫小狗都从这个"模板"造出来，统一管理。

> 💡 **类（class）** = 模板（饼干模具），**实例（instance）** = 用模板生产的东西（饼干）。

## 2. 第一个类：定义、`self`、`__init__` 🐾

```python
class Pet:
    def __init__(self, name, age):     # 构造方法（造饼干时填模具）
        self.name = name               # self = 当前实例
        self.age = age

    def bark(self):                    # 实例方法
        print(f"{self.name} 在叫！")

dog = Pet("旺财", 3)                    # 调用 __init__
dog.bark()                             # 旺财 在叫！
print(dog.name, dog.age)               # 旺财 3
```

**逐行**：
- `class Pet:` 定义一个名为 Pet 的类。约定**首字母大写**。
- `__init__` 是**初始化方法**，对象一造出来就调用，给它"装属性"。
- `self` 永远是第一个参数，代表"我自己这个实例"。
- `dog = Pet("旺财", 3)` 实例化（不需要写 `new`）。

## 3. 三种方法：实例 / 类 / 静态 ⚙️

```python
class Counter:
    total = 0                              # 类属性（所有实例共享）

    def __init__(self):
        self.value = 0                     # 实例属性

    def incr(self):                        # 实例方法
        self.value += 1
        Counter.total += 1

    @classmethod
    def stats(cls):                        # 类方法（cls = 类本身）
        print(f"总共 {cls.total} 次")

    @staticmethod
    def help():                            # 静态方法（没有 self / cls）
        print("Counter 用法：c.incr()")

a = Counter(); a.incr(); a.incr()
b = Counter(); b.incr()
Counter.stats()      # 总共 3 次
Counter.help()
```

| 方法 | 第一个参数 | 何时用 |
|------|-----------|--------|
| 实例方法 | `self` | 操作单个实例 |
| 类方法 | `cls` | 操作类本身（如工厂方法） |
| 静态方法 | 无 | 工具函数，逻辑上属于类 |

## 4. 公有 / 私有：下划线约定 🔒

```python
class Account:
    def __init__(self, owner, balance):
        self.owner = owner          # 公有
        self._secret = "内部用"      # 单下划线：约定"别动我"
        self.__pin = "1234"         # 双下划线：名称改写

a = Account("李雷", 100)
print(a.owner)              # 李雷
print(a._secret)            # 仍然能访问，只是约定
# print(a.__pin)            # ❌ AttributeError
print(a._Account__pin)      # 1234（双下划线被改成 _类名__name）
```

> 💡 Python **没有真正的私有** — 只有"君子协定"。`_` 表示"内部"，`__` 表示"我真的不想让你碰"。

## 5. 继承：is-a 关系 🧬

```python
class Animal:
    def __init__(self, name):
        self.name = name
    def eat(self):
        print(f"{self.name} 在吃饭")

class Dog(Animal):                           # Dog "是一种" Animal
    def __init__(self, name, breed):
        super().__init__(name)               # 调用父类 __init__
        self.breed = breed
    def bark(self):
        print(f"{self.name}（{self.breed}）汪汪汪")

d = Dog("旺财", "柴犬")
d.eat()      # 旺财 在吃饭     ← 继承来的
d.bark()     # 旺财（柴犬）汪汪汪
```

> 💡 `super()` 是"找父类"的快捷方式，避免硬编码父类名。

## 6. 多继承与 MRO 🪜

**问题**：如果一个类继承了两个父类，调用方法该走哪条路？答：按 **MRO（方法解析顺序）**，由 **C3 算法** 决定。

```python
class A:
    def hi(self): print("A")
class B(A):
    def hi(self): print("B")
class C(A):
    def hi(self): print("C")
class D(B, C):
    pass

D().hi()                  # B  按 MRO 找第一个有 hi 的
print(D.__mro__)
# (D, B, C, A, object)
```

**简单理解 C3**：从左往右深度优先，但保证子类总在父类之前 — 不会出现"继承了 B 但走 A"。

> 💡 多继承容易乱，能不用就不用。可以用 **mixin**（专门提供一组方法的小类）来组合。

## 7. 多态：同一个调用，不同表现 🎭

```python
class Cat(Animal):
    def speak(self): print("喵")
class Dog(Animal):
    def speak(self): print("汪")

for pet in [Cat("咪咪"), Dog("旺财")]:
    pet.speak()           # 喵 / 汪 — 同样的 speak()，不同行为
```

> 💡 多态让代码"对接口编程，不对实现编程"，扩展性强。

## 8. 魔术方法（dunder）✨

| 方法 | 触发时机 |
|------|---------|
| `__init__` | 创建实例 |
| `__repr__` | `repr(x)` / 调试输出 |
| `__str__` | `print(x)` / `str(x)` |
| `__eq__` | `x == y` |
| `__hash__` | 用作 dict 键 / set 元素 |
| `__len__` | `len(x)` |
| `__getitem__` | `x[i]` |
| `__setitem__` | `x[i] = v` |
| `__call__` | `x()` 把实例当函数用 |
| `__add__` | `x + y` |

```python
class Vector:
    def __init__(self, x, y):
        self.x, self.y = x, y
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"
    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)
    def __eq__(self, other):
        return (self.x, self.y) == (other.x, other.y)
    def __len__(self):
        return 2
    def __getitem__(self, i):
        return (self.x, self.y)[i]
    def __call__(self, scale):
        return Vector(self.x*scale, self.y*scale)

v1 = Vector(1, 2); v2 = Vector(3, 4)
print(v1 + v2)        # Vector(4, 6)
print(v1 == Vector(1, 2))   # True
print(len(v1))        # 2
print(v1[0], v1[1])   # 1 2
print(v1(10))         # Vector(10, 20)
```

> 💡 写完魔术方法，类就**像内置类型一样自然**。

## 9. `@property`：把方法变属性 🪄

```python
class Circle:
    def __init__(self, r):
        self._r = r

    @property
    def radius(self):
        return self._r

    @radius.setter
    def radius(self, value):
        if value < 0:
            raise ValueError("半径不能为负")
        self._r = value

    @property
    def area(self):
        return 3.14159 * self._r ** 2

c = Circle(3)
print(c.radius)    # 3       — 不用括号！
print(c.area)      # 28.27...
c.radius = 5       # 赋值时会进 setter，自动校验
# c.radius = -1    # ❌ ValueError
```

> 💡 `@property` 让你**先写公开属性，再加校验**，不破坏调用代码。

## 10. dataclass：少写 90% 模板 🪶

```python
from dataclasses import dataclass

@dataclass
class Point:
    x: int
    y: int = 0           # 默认值

p1 = Point(1, 2)
p2 = Point(1, 2)
print(p1)            # Point(x=1, y=2)   ← 自动 __repr__
print(p1 == p2)      # True              ← 自动 __eq__
```

`@dataclass` 自动生成 `__init__` / `__repr__` / `__eq__`，专为"数据载体类"设计。

```python
@dataclass(frozen=True)        # frozen=True → 不可变
class Money:
    amount: float
    currency: str = "CNY"
```

## 11. enum：枚举值 🚥

```python
from enum import Enum

class Color(Enum):
    RED = 1
    GREEN = 2
    BLUE = 3

c = Color.RED
```

## 12. 实战 1：完整 Vector 类 ➕

```python
import math
from dataclasses import dataclass

@dataclass
class Vec:
    x: float
    y: float

    def __add__(self, other):
        return Vec(self.x + other.x, self.y + other.y)
    def __sub__(self, other):
        return Vec(self.x - other.x, self.y - other.y)
    def __mul__(self, k):
        return Vec(self.x * k, self.y * k)
    def length(self):
        return math.hypot(self.x, self.y)

a = Vec(1, 2); b = Vec(3, 4)
print(a + b)       # Vec(x=4, y=6)
print((b - a).length())   # 2.828...
print(a * 3)       # Vec(x=3, y=6)
```

## 13. 实战 2：银行账户体系 🏦

```python
class Account:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self._balance = balance

    def deposit(self, n):
        self._balance += n

    def withdraw(self, n):
        if n > self._balance:
            raise ValueError("余额不足")
        self._balance -= n

    @property
    def balance(self):
        return self._balance

    def __repr__(self):
        return f"<{type(self).__name__} {self.owner} ¥{self._balance}>"

class SavingsAccount(Account):
    def __init__(self, owner, balance=0, rate=0.03):
        super().__init__(owner, balance)
        self.rate = rate
    def add_interest(self):
        self._balance *= (1 + self.rate)

class CheckingAccount(Account):
    OVERDRAFT = -1000
    def withdraw(self, n):           # 重写：允许透支 1000
        if n > self._balance - self.OVERDRAFT:
            raise ValueError("超出透支限额")
        self._balance -= n

s = SavingsAccount("李雷", 1000)
s.add_interest()
print(s)                              # <SavingsAccount 李雷 ¥1030.0>

c = CheckingAccount("韩梅梅", 500)
c.withdraw(800)                       # 允许透支
print(c)                              # <CheckingAccount 韩梅梅 ¥-300>
```

## 14. 实战 3：扑克牌发牌器 🃏

```python
import random
from dataclasses import dataclass

@dataclass(frozen=True)
class Card:
    rank: str
    suit: str
    def __repr__(self): return f"{self.rank}{self.suit}"

class Deck:
    RANKS = list("23456789TJQKA")
    SUITS = "♠♥♦♣"

    def __init__(self):
        self.cards = [Card(r, s) for r in self.RANKS for s in self.SUITS]
        random.shuffle(self.cards)

    def deal(self, n):
        hand, self.cards = self.cards[:n], self.cards[n:]
        return hand

    def __len__(self): return len(self.cards)

deck = Deck()
print(deck.deal(5))     # 例：[8♥, A♠, 3♣, ...]
print("剩余", len(deck))
```

## 🐛 常见错误集合

| 错误 | 原因 |
|------|------|
| 忘写 `self` | 实例方法第一个参数必须是 self |
| `__init__` 没写 `self.x =` | 属性没绑到实例 |
