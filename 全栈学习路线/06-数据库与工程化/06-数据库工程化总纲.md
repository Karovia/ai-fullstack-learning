# 第 06 章 · 数据库 + 工程化（6 周）

## 🎯 章节目标

- 精通 PostgreSQL（SQL + 索引 + 事务 + 锁）
- 掌握 Redis（5 种数据结构 + 持久化 + 集群）
- 会用 ORM（SQLAlchemy 2.0 / Drizzle）+ 不依赖 ORM 写原生 SQL
- 向量数据库（pgvector / Qdrant）—— 为 AI Agent 章节准备
- Docker / Docker Compose / Kubernetes 基础
- CI/CD + 监控 + 日志的完整工程链路

## 📅 周计划

| 周 | 主题 | 产出 |
|---|------|------|
| W1 | SQL 基础 + PostgreSQL 进阶 | 50 道 SQL 题 |
| W2 | 索引 / 事务 / 锁 / 性能优化 | 慢查询调优案例 |
| W3 | Redis 五大数据结构 + 应用场景 | Redis 实战项目 |
| W4 | ORM：SQLAlchemy 2.0 + Alembic 迁移 | 重构 05 章项目 |
| W5 | 向量数据库 + pgvector + 全文搜索 | 语义搜索 demo |
| W6 | Docker + K8s + CI/CD + 监控 | 生产级部署 |

## 📖 详细内容

### W1 - SQL & PostgreSQL
- SELECT / INSERT / UPDATE / DELETE
- JOIN（INNER / LEFT / RIGHT / FULL）
- GROUP BY / HAVING / 窗口函数
- 子查询 + CTE（WITH 语句，必学）
- PostgreSQL 特性：JSON / JSONB / Array / UUID
- psql 命令行 + DBeaver 客户端

### W2 - 索引、事务、锁
- B-tree / Hash / GIN / GiST 索引选择
- EXPLAIN ANALYZE 看执行计划
- 事务 ACID + 隔离级别（4 种）
- 行锁 vs 表锁 vs 死锁
- MVCC 原理
- 性能调优套路（pg_stat_statements）

### W3 - Redis
- string / list / hash / set / zset
- 高级：Stream / HyperLogLog / Bitmap / Geo
- 持久化：RDB vs AOF
- 主从 / 哨兵 / Cluster
- 应用：缓存 / 排行榜 / 限流 / 分布式锁 / 消息队列

### W4 - ORM
- SQLAlchemy 2.0（新风格 + 异步）
- Alembic 数据库迁移
- 关系：one-to-many / many-to-many / 多态
- N+1 查询问题 + 解决（selectinload / joinedload）
- Drizzle ORM（Node.js 现代选择）

### W5 - 向量数据库（AI 准备）
- 嵌入向量（embedding）原理
- pgvector：在 PostgreSQL 中存向量
- Qdrant / Weaviate / Milvus 对比
- ANN 算法（HNSW / IVF）
- 全文搜索：PG full-text / Elasticsearch / Meilisearch
- 混合搜索（向量 + 关键词 + 重排）

### W6 - 工程化与部署
- Docker 进阶（多阶段、buildkit、缓存）
- docker-compose 编排
- Kubernetes 基础（Pod / Deployment / Service / Ingress）
- Helm Chart
- 监控：Prometheus + Grafana + Loki + Tempo（grafana 全家桶）
- Sentry 错误追踪

## 🚀 章节大项目：`mini-twitter` 高并发短文社区

### 需求
- 用户、关注、推文、点赞、转发、@提及、#话题
- 时间线：自己 + 关注的人，按时间倒序
- 热门排行榜（每分钟更新）
- 全文搜索 + 语义搜索
- 5 万用户 / 100 万推文规模数据
- 单接口 P95 < 100ms

### 技术栈
- 后端：FastAPI（05 章学过）
- DB：PostgreSQL 16 + pgvector
- 缓存：Redis 7
- 队列：ARQ
- 搜索：Meilisearch（轻量替代 ES）
- 监控：Grafana + Prometheus + Loki
- 部署：Docker Compose（学习）+ Kubernetes（进阶）

### 验收标准
- [ ] 数据库 schema 设计文档完整（ER 图）
- [ ] 索引覆盖所有热点查询，pg_stat_statements 看不到慢 SQL
- [ ] 时间线接口用 Redis 缓存，命中率 > 90%
- [ ] 用 [k6](https://k6.io/) 压测：1000 RPS 持续 5 分钟，P95 < 100ms
- [ ] 全文 + 向量混合搜索可用
- [ ] 监控大屏展示：QPS / 延迟 / 错误率 / 资源使用
- [ ] 一个 commit / docker-compose up 即可启动整套

### 进阶挑战

- 用 ClickHouse 做实时分析
- 推文按用户分库分表（10 个 shard）
- 接入 OpenTelemetry 全链路追踪
