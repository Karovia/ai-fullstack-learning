# W01 - JavaScript 语法与类型

> 本周目标：把 JS 当 Python 的"远房表哥"看待——长得像但脾气怪。会写第一行 JS、理解 7 种原始类型、看懂 `==` 的坑、能用数组对象模板字符串写小程序。

## 0. 一段简史（5 分钟读懂）

- **1995**：Brendan Eich 用 10 天写出 JS（最初叫 Mocha → LiveScript → JavaScript，蹭 Java 热度）
- **2009 ES5**：严格模式 / JSON / Array.map 等
- **2015 ES6 (ES2015)**：里程碑！`let/const` / 箭头函数 / class / Promise / 模板字符串
- **2017 async/await**
- **2020+ ES2020-2024**：可选链 `?.`、空值合并 `??`、顶层 await、`#` 私有字段……
- **现状**：每年 6 月发新版本，本教程基于 ES2024+

> 类比：Python 也有 2→3 的大跨越，JS 的 ES5→ES6 类似。本教程**只教 ES6+**，老语法（var、function 表达式继承）只在"为什么不用"时提及。

## 1. 运行环境：浏览器 vs Node.js

| 维度 | 浏览器 | Node.js |
|------|--------|---------|
| 用途 | 操控网页 DOM | 写后端、命令行工具 |
| 全局对象 | `window` | `global` / `globalThis` |
| 输入输出 | `alert/console.log/prompt` | `console.log` + `process.stdin` |
| 模块 | ES Module `<script type=module>` | CommonJS `require` 或 ESM |
| 文件 IO | 没有（沙盒） | 有 `fs` 模块 |

> 类比：Python 只有一个解释器，JS 有"两个家"。同一段 `console.log(1+1)` 两边都能跑，但 `document.body` 只在浏览器有，`fs.readFile` 只在 Node 有。

## 2. 三种"第一个 JS 程序"

### 方式 A：浏览器 Console（最快）
打开 Chrome → F12 → Console 选项卡 → 输入：

```javascript
console.log('Hello, JS!');  // 输出: Hello, JS!
1 + 1                       // 直接显示 2
```

### 方式 B：HTML 文件
新建 `hello.html`：

```html
<!DOCTYPE html>
<html>
<body>
  <h1 id="title">未变</h1>
  <script>
    document.getElementById('title').textContent = 'JS 改了我！';
    console.log('页面加载完成');
  </script>
</body>
</html>
```

双击打开，标题会变成 "JS 改了我！"。

### 方式 C：Node.js
新建 `hello.js`：

```javascript
console.log('Hello from Node');
console.log(`Node 版本: ${process.version}`);
```

终端运行：`node hello.js`

## 3. 变量声明：`const` 优先，`let` 次之，`var` 禁用

```javascript
const PI = 3.14;       // 常量，不能再赋值
let count = 0;         // 可变变量
count = count + 1;     // OK
// PI = 3.15;          // ❌ TypeError

var oldStyle = '别用';  // ❌ 禁用：函数作用域 + 提升导致一堆坑
```

| 关键字 | 作用域 | 可重新赋值 | 提升 | 推荐 |
|--------|--------|-----------|------|------|
| `const` | 块级 `{}` | ❌（对象内部可改） | 有但 TDZ | ✅ 默认 |
| `let` | 块级 | ✅ | 有但 TDZ | ✅ 需要变 |
| `var` | 函数级 | ✅ | 提升为 undefined | ❌ |

> **类比 Python**：Python 没有 const，`PI = 3.14` 全靠自觉大写表示常量。JS 有真正的常量保护——`const` 锁住引用（但对象内部属性还能改，因为锁的是"门牌号"不是"屋里东西"）。

```javascript
const user = { name: 'Tom' };
user.name = 'Jerry';   // ✅ 改屋里的人
// user = { name: 'X' }; // ❌ 换门牌号，禁止
```

## 4. 7 种原始类型（Primitive Types）

```javascript
typeof 42              // 'number'    数字（整浮共用）
typeof 'hi'            // 'string'    字符串
typeof true            // 'boolean'   布尔
typeof undefined       // 'undefined' 未定义
typeof null            // 'object'    ⚠️ 历史 Bug！实际是 null 类型
typeof Symbol('x')     // 'symbol'    ES6 唯一标识
typeof 9007199254740993n // 'bigint'  大整数（末尾 n）
typeof {}              // 'object'    对象（非原始）
typeof []              // 'object'    数组也是 object
typeof function(){}    // 'function'  函数（特殊 object）
```

> **类比 Python**：Python 有 `int / float / str / bool / None / bytes...`，JS 把整数浮点统一为 `number`（`0.1 + 0.2 !== 0.3` 这个浮点坑两边都有）。`null` 类似 `None`，但 JS 还多一个 `undefined`（"我都没赋值过"）。

### `null` vs `undefined` 的微妙区别

```javascript
let a;            // 声明没赋值 → undefined
let b = null;     // 主动表示"空" → null
console.log(a == b);   // true  （宽松相等，特例）
console.log(a === b);  // false （严格相等，类型不同）
```

约定：函数没返回值 → `undefined`；你想表达"故意为空" → `null`。

## 5. 隐式转换：JS 头号坑

JS 会偷偷做类型转换，这是 Python 不会做的（Python `'1' + 1` 直接报错，JS 给你 `'11'`）。

```javascript
'5' + 3        // '53'   字符串拼接
'5' - 3        // 2      减法触发转数字
true + 1       // 2      true→1
[] + []        // ''     空数组转空字符串
[] + {}        // '[object Object]'
```

### 经典坑题：`[] == ![]` 为什么是 `true`？

```javascript
[] == ![]      // true ?!
```

拆解：
1. `![]` → `!true` → `false`（空数组转 boolean 为 true，取反 false）
2. 变成 `[] == false`
3. 宽松相等遇到 boolean，先转 number：`[] == 0`
4. `[]` 转 number：先 `toString()` → `''`，再 `Number('')` → `0`
5. `0 == 0` → `true`

> 教训：**永远用 `===`**，本教程从此告别 `==`。

## 6. `===` 严格相等（必背）

```javascript
1 === '1'       // false  类型不同
1 === 1         // true
NaN === NaN     // false  ⚠️ NaN 不等于自己！用 Number.isNaN(x) 检测
0 === -0        // true
```

## 7. 模板字符串

```javascript
const name = 'Tom', age = 18;
const s = `我叫 ${name}，今年 ${age + 1} 岁`;
// 多行无需 \n
const html = `
  <div>
    <p>${name}</p>
  </div>
`;
```

> 类比 Python f-string：`f'我叫 {name}'` ↔ `` `我叫 ${name}` ``。

## 8. 数组（Array）

```javascript
const arr = [1, 2, 3, 'mix', true];

arr.push(4);          // 末尾加 → [1,2,3,'mix',true,4]
arr.pop();            // 末尾删
arr.unshift(0);       // 头部加
arr.shift();          // 头部删
arr.slice(1, 3);      // 截取 [2,3]，不改原数组
arr.splice(1, 2);     // 删除 index=1 起 2 个，改原数组

// 高阶函数（W02 详讲）
[1,2,3].map(x => x * 2);           // [2,4,6]
[1,2,3,4].filter(x => x % 2 === 0); // [2,4]
[1,2,3].reduce((a, b) => a + b, 0); // 6
```

> 类比 Python list：`append/pop/insert/del/[1:3]` 一一对应。`map/filter` 类似 Python 的列表推导和 `filter()`。

## 9. 对象字面量 `{}`（≈ Python dict）

```javascript
const user = {
  name: 'Tom',
  age: 18,
  greet() { return `hi, ${this.name}`; },  // 方法简写
};

user.name;          // 'Tom'   点访问
user['age'];        // 18      键访问
user.email = 'x@y'; // 添加属性
delete user.age;    // 删除
'name' in user;     // true
Object.keys(user);  // ['name','greet','email']
```

## 10. JSON

```javascript
const obj = { name: 'Tom', tags: ['a', 'b'] };
const str = JSON.stringify(obj);   // '{"name":"Tom","tags":["a","b"]}'
const back = JSON.parse(str);      // 还原对象
```

> JSON 不允许：单引号、尾随逗号、注释、function、undefined。

## 11. 严格模式 `'use strict'`

文件首行加上 `'use strict';` 或用 ES Module（自动严格）。变量必须声明、`this` 不再指 window 等，少踩坑。

### 练习 3：迷你计算器
```javascript
function calc(a, op, b) {
  const ops = {
    '+': (x, y) => x + y,
    '-': (x, y) => x - y,
    '*': (x, y) => x * y,
    '/': (x, y) => y === 0 ? NaN : x / y,
  };
  return ops[op]?.(a, b) ?? '不支持的运算符';
}
console.log(calc(3, '+', 4)); // 7
console.log(calc(10, '/', 0)); // NaN
```

## 常见错误

1. **`var` 提升**：`console.log(x); var x = 1;` 不报错却得 `undefined`——禁用即可
2. **`typeof null === 'object'`**：判 null 用 `x === null`
3. **`NaN === NaN` 是 false**：用 `Number.isNaN(x)`
4. **数组判断**：`typeof [] === 'object'`，应使用 `Array.isArray(x)`
5. **JSON 不能含函数和 undefined**
