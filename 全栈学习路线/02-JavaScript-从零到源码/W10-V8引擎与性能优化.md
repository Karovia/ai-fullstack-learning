> 💡 **本周目标**：揭开 JS 引擎的"黑箱"——V8 怎么把动态脚本跑出接近 C++ 的速度，浏览器怎么把代码变成像素，以及如何让你的应用真的"快"。第 02 章压轴！

## 🏗️ 一、V8 整体架构

> 📖 **类比**：你写一份菜谱（JS 代码），V8 像一家中央厨房：
> - **Parser** = 看菜谱（解析成 AST）
> - **Ignition** = 学徒厨师（解释执行 + 收集"哪道菜常做"）
> - **TurboFan / Maglev** = 大厨（把高频菜直接编译成机器码，飞快）

```
源码
 │
 ▼
[Parser/Scanner] ── 词法/语法分析 ──> AST + Scope
 │
 ▼
[Ignition] ── 字节码解释器 + 类型反馈 ──> 慢但通用
 │
 ▼ (热点函数)
[Maglev] ── 中等优化（V8 11+）
 │
 ▼ (更热)
[TurboFan] ── 激进优化 + 假设
 │
 ▼ (假设错了)
反优化（Deopt）回到 Ignition
```

### 关键概念

- **JIT (Just-In-Time)**：边运行边编译，不是先编译再运行
- **类型反馈**：Ignition 收集"这个变量都是 number 吗"
- **假设 + 守卫**：TurboFan 编译时大胆假设；运行时一次次校验
- **Deopt**：假设错了就丢弃机器码，退回字节码

## 🆔 二、Hidden Class（隐藏类）

> 📖 **类比**：JS 对象表面"自由"，V8 偷偷给"长得一样的对象"分配同一个"形状档案"，访问属性时走快路径。

```javascript
// ✅ 好：两个对象共享同一个 hidden class
function Point(x, y) {
  this.x = x;
  this.y = y;
}
const a = new Point(1, 2);
const b = new Point(3, 4);

// ❌ 坏：c 多了一个属性，hidden class 变了
const c = new Point(5, 6);
c.z = 7;

// ❌ 更坏：属性顺序不同，hidden class 不同！
const d = { x: 1, y: 2 };
const e = { y: 2, x: 1 };  // 形状和 d 不一样
```

### V8 友好准则

1. **构造函数里一次性把属性初始化完**
2. **顺序一致**
3. **类型稳定**（`x` 始终是 number，别一会 string）
4. **避免 `delete obj.x`**（会破坏 hidden class）

---

## ⚡ 三、Inline Cache（内联缓存）

每次访问 `obj.x`，V8 第一次记下"这个 hidden class 的 x 在偏移量 8"，下次直接读 → 接近 C++ 速度。

```javascript
function getX(p) { return p.x; }
// 调 1000 次，全是同形状 Point → IC 命中，快
// 突然传一个 { x: 1, y: 2, z: 3 } → IC 失效（变多态/巨态）
```

| IC 状态 | 说明 |
|---|---|
| Monomorphic（单态） | 1 种 hidden class，最快 |
| Polymorphic（多态） | 2-4 种，还行 |
| Megamorphic（巨态） | 5+ 种，慢 |

## 🗑️ 四、垃圾回收（GC）

> 📖 **类比**：内存像图书馆，GC 是管理员，定期把"没人借了"的书回收。

### 标记-清除 + 分代

V8 把堆分两代：

| 代 | 特点 | GC 算法 |
|---|---|---|
| **新生代**（Young）| 小、新对象多、活不长 | Scavenge（复制半区）|
| **老生代**（Old）| 大、活下来的"老人" | Mark-Sweep + Mark-Compact |

新对象先进新生代；活过 2 次 GC → 晋升到老生代。

### 内存泄漏常见来源

```javascript
// 1. 全局变量（忘了 var/let/const）
function f() { leak = 'oops'; }   // → window.leak

// 2. 被遗忘的定时器
const id = setInterval(() => use(bigData), 1000);
// 没 clearInterval，bigData 永远活着

// 3. DOM 引用
const cache = [];
cache.push(document.querySelector('#removed'));
// 节点已从 DOM 移除，但 cache 还在引用 → 内存里还活着

// 4. 闭包不当
function make() {
  const huge = new Array(1e6);
  return () => huge[0];   // huge 被闭包困住
}
```

> ✅ **武器**：用 `WeakMap` / `WeakRef` 持有可被 GC 的引用；及时 `clearTimeout`、移除事件监听。

## 🎨 六、浏览器渲染管线

```
HTML  ──Parse──> DOM 树
CSS   ──Parse──> CSSOM 树
                  │
                  ▼ 合并
              Render Tree
                  │
                  ▼
              Layout（布局/重排）— 算位置和大小
                  │
                  ▼
              Paint（绘制/重绘）— 算每个像素颜色
                  │
                  ▼
              Composite（合成）— GPU 把图层贴一起
```

### 重排（Reflow） vs 重绘（Repaint）

| 操作 | 触发 |
|---|---|
| 改 width/height/top/left | **重排** + 重绘 + 合成 |
| 改 background-color/color | 重绘 + 合成 |
| 改 transform/opacity | **只合成**（最便宜！）|

> ✅ **优化建议**：动画用 `transform` + `opacity`，避免改触发 layout 的属性。

### 关键渲染路径优化

1. **CSS 放头部**（不阻塞 DOM 但阻塞渲染，越早下载越好）
2. **JS 加 `defer` / `async`**（不阻塞解析）
3. **预连接关键域名**：`<link rel="preconnect" href="...">`
4. **图片加 `loading="lazy"`**
5. **关键 CSS 内联**

## 📏 七、Web Vitals（核心性能指标）

| 指标 | 含义 | 目标 |
|---|---|---|
| **LCP** Largest Contentful Paint | 最大内容绘制 | < 2.5s |
| **FID** First Input Delay | 首次输入延迟（已弃用） | < 100ms |
| **INP** Interaction to Next Paint | 交互响应（FID 接班人）| < 200ms |
| **CLS** Cumulative Layout Shift | 累计布局偏移 | < 0.1 |
| **TTFB** Time to First Byte | 首字节时间 | < 800ms |
| **FCP** First Contentful Paint | 首次内容绘制 | < 1.8s |

```javascript
// web-vitals 库测量
import { onLCP, onCLS, onINP } from 'web-vitals';
onLCP(console.log);
onCLS(console.log);
onINP(console.log);
```

### 提升 LCP

- 服务端响应快（CDN、缓存）
- 关键图片用 `<link rel="preload" as="image">`
- 字体加 `font-display: swap`

### 降低 CLS

- 图片/iframe 一定写 `width` `height`
- 动态插入元素留好占位高度

## 🧹 八、写"快代码"的 10 条军规

1. **避免动态属性增删** → 保持 hidden class 稳定
2. **类型一致** → 别一会 number 一会 string
3. **for 循环优于 forEach**（热点）
4. **避免 `try/catch` 包大块代码**（影响 V8 优化）
5. **少用 `arguments`**，用 rest `(...args)`
6. **批量 DOM 操作**：用 `DocumentFragment`，一次性 append
7. **读写分离**：连续读（`offsetHeight`）放一起，再连续写
8. **长列表虚拟滚动**
9. **图片懒加载 + 合适尺寸 + WebP/AVIF**
10. **节流 / 防抖 滚动、resize、input**

```javascript
// ❌ 触发 N 次重排
for (let i = 0; i < 100; i++) {
  list.appendChild(createItem(i));
}

// ✅ 1 次重排
const frag = document.createDocumentFragment();
for (let i = 0; i < 100; i++) frag.appendChild(createItem(i));
list.appendChild(frag);
```

## 🐛 常见错误

| 现象 | 原因 |
|---|---|
| 首屏卡顿 | 同步 JS 太多 / LCP 图片太大 |
| 滚动掉帧 | scroll 没节流 + 触发 layout |
| 内存涨不停 | 闭包 / 定时器 / 事件未清 |
| 动画卡顿 | 用了 left/top 而非 transform |
| INP 偏高 | 主线程长任务（>50ms）阻塞输入 |
