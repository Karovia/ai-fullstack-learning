# W08 - 模块系统与工程化 📦

> 💡 **本周目标**：把零散的 JS 文件组织成可维护、可发布、可复用的工程。从"写脚本"升级到"做项目"。

## 🕰️ 一、模块的"进化史"

> 📖 **类比**：从一个人在一张纸上乱写（全局变量），到大家用 USB 互传文件（IIFE），到云盘共享（CommonJS），最后到现代浏览器/Node 都认的"国际标准"（ESM）。

| 时代 | 方案 | 特点 |
|---|---|---|
| 远古 | 全局变量 | 命名冲突地狱 |
| 2010 | IIFE | 立即执行函数，闭包隔离 |
| 2011 | AMD（RequireJS）| 浏览器异步加载 |
| 2012 | CMD（SeaJS）| 阿里玉伯出品 |
| 2009 | CommonJS | Node.js 同步加载 |
| 2014 | UMD | 一份代码兼容多种 |
| 2015 | **ESM** | 官方标准，浏览器/Node 通用 |

## 📜 二、CommonJS（Node.js 经典）

```javascript
// math.js
function add(a, b) { return a + b; }
module.exports = { add };
// 或：exports.add = add;

// app.js
const { add } = require('./math');
console.log(add(1, 2));
```

> ⚠️ `exports` 只是 `module.exports` 的别名引用。如果直接 `exports = {...}` 会断开关系，导出的依然是原对象。**记住：永远赋值给 `module.exports`**。

特点：

- **同步加载**（require 立即返回）
- 值是**拷贝**（导出基本类型后改不会影响）
- 可以**条件 require**

## 🌐 三、ESM（现代标准）

```javascript
// math.js
export function add(a, b) { return a + b; }
export const PI = 3.14;
export default class Math2 {}

// app.js
import Math2, { add, PI } from './math.js';
import * as math from './math.js';
import { add as plus } from './math.js';  // 重命名

// 动态导入（返回 Promise）
const m = await import('./math.js');
```

### CJS vs ESM 关键差异

| 维度 | CommonJS | ESM |
|---|---|---|
| 加载 | 同步 | 异步（动态时） |
| 值 | 拷贝 | **绑定**（引用） |
| `this` | `module.exports` | `undefined` |
| `__filename/__dirname` | 有 | 无（用 `import.meta.url`） |
| top-level await | 不支持 | ✅ |
| Tree shaking | 难 | ✅ 静态分析友好 |

### 互操作（最容易踩坑的地方）

```javascript
// CJS 引 ESM：必须用 await import()
const esm = await import('./esm-only.mjs');

// ESM 引 CJS：默认导出 = module.exports 整体
import pkg from 'lodash';      // = require('lodash')
const { debounce } = pkg;
// 或部分 named exports 由 Node 静态分析提供
import { debounce } from 'lodash';
```

> 💡 **package.json 怎么标识**：`"type": "module"` 让 `.js` 视为 ESM；不加默认 CJS。`.mjs` 强制 ESM；`.cjs` 强制 CJS。

## 🌳 四、Tree Shaking（摇掉死代码）

```javascript
// utils.js
export const used = () => 1;
export const unused = () => 2;

// app.js
import { used } from './utils.js';
console.log(used());
```

构建工具（Vite/Rollup/webpack 5）会**静态分析**，把 `unused` 从最终 bundle 删掉。

**前提**：

1. 用 ESM（CJS 是动态的，分析不了）
2. 没有副作用，或在 `package.json` 标 `"sideEffects": false`

## 📋 六、package.json 全字段

```json
{
  "name": "my-pkg",
  "version": "1.2.3",          // semver: 主版本.次.补丁
  "type": "module",             // 文件默认 ESM
  "main": "./dist/index.cjs",  // CJS 入口（老）
  "module": "./dist/index.mjs",// ESM 入口（webpack 用）
  "types": "./dist/index.d.ts",
  "exports": {                  // 现代多入口（最重要！）
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.mjs",
      "require": "./dist/index.cjs"
    },
    "./utils": "./dist/utils.mjs"
  },
  "files": ["dist"],            // 发布到 npm 的文件
  "scripts": {
    "build": "tsup",
    "test": "vitest"
  },
  "dependencies": {},
  "devDependencies": {},
  "peerDependencies": {},
  "sideEffects": false          // 帮助 tree-shaking
}
```

### semver 语义化版本

```
1.2.3
│ │ └── patch：bugfix
│ └──── minor：新增功能（兼容）
└────── major：破坏性更新

^1.2.3 → 兼容更新（1.x.x 内最新）
~1.2.3 → 仅 patch（1.2.x）
1.2.3  → 锁死
```

### 锁文件作用

`package-lock.json` / `pnpm-lock.yaml` 记录**精确版本 + 依赖树**，保证团队和服务器装出一样的包。**必须提交到 git**。

## 🔧 九、手写 mini require（CJS 简化版）

```javascript
const fs = require('fs');
const path = require('path');
const vm = require('vm');

const cache = {};

function myRequire(filePath) {
  const abs = path.resolve(filePath);
  if (cache[abs]) return cache[abs].exports;

  const module = { exports: {} };
  cache[abs] = module;  // ← 先放进缓存解决循环依赖

  const code = fs.readFileSync(abs, 'utf8');
  const wrapper = `(function(exports, require, module, __filename, __dirname) { ${code} })`;
  const fn = vm.runInThisContext(wrapper);
  fn(module.exports, myRequire, module, abs, path.dirname(abs));

  return module.exports;
}
```

> 💡 **循环依赖**：A 引 B，B 又引 A。Node 的解决方案就是上面那行——先把"未完成的 module.exports"放缓存，循环回来时拿到的是当前已导出的部分。

## 🐛 常见错误

| 现象 | 原因 |
|---|---|
| `require is not defined` in ESM | ESM 没有 require，用 `createRequire` |
| `__dirname is not defined` | ESM 用 `import.meta.url` + `fileURLToPath` |
| 发包后 import 不到子路径 | `exports` 字段没加 `"./xx"` |
| 循环依赖结果是 undefined | 重构或推迟取值 |
| pnpm 找不到依赖 | 没声明就直接用是不行的（严格模式） |
