# W03 - 闭包与作用域

> 本周目标：彻底理解 JS 的作用域机制和闭包，看穿一切"`for + setTimeout`"面试题。结束时能手写 `once` / `curry` / 防抖节流。

## 1. 词法作用域 vs 动态作用域

**词法作用域（lexical scope）**：变量的可见范围**在写代码时就决定了**，看代码在哪里就行。JS 是词法作用域。

**动态作用域**：变量看运行时调用栈。Bash 的变量、早期 Lisp 是动态作用域。

```javascript
const x = 10;
function foo() { console.log(x); }   // x 在 foo 定义处的外层 → 全局 10
function bar() {
  const x = 99;
  foo();                             // 仍然打印 10（词法），不是 99
}
bar();
```

> **类比 Python**：Python 也是词法作用域，所以这条直觉是通用的。

## 2. 三种作用域

| 作用域 | 关键字 | 示例 |
|--------|--------|------|
| 全局 | 顶层 | `const A = 1;` 在所有地方都看得见 |
| 函数 | `function() {}` 内 | `var/let/const` 都受限 |
| 块级 | `{ }`（if/for/单纯花括号） | 仅 `let/const` 受限，`var` 穿透 |

```javascript
{
  let a = 1;
  var b = 2;
}
// console.log(a); // ReferenceError
console.log(b);    // 2  ⚠️ var 穿透块级
```

> Python 没有块级作用域（`if` 里定义的变量外面也能用），JS 的 `let/const` 引入了块级，更接近其他主流语言。

## 3. 变量提升 hoisting

JS 编译时会把"声明"提到作用域顶部。

```javascript
console.log(x);   // undefined（不是 ReferenceError）
var x = 5;
// 等价于
var x;
console.log(x);   // undefined
x = 5;

console.log(foo());  // 'hi'
function foo() { return 'hi'; }   // 函数声明整个被提升

// 函数表达式不提升
console.log(bar);    // undefined
var bar = function() {};
```

## 4. 暂时性死区 TDZ（`let/const`）

```javascript
console.log(y);   // ReferenceError: Cannot access 'y' before initialization
let y = 1;
```

`let/const` **也提升**，但在声明前访问会报错——这段"被提升但还没到声明行"的区域叫 **暂时性死区（Temporal Dead Zone）**。设计目的：强制写代码顺序合理。

## 5. 执行上下文 + 调用栈

每次调用函数，JS 引擎会创建一个 **执行上下文（Execution Context）** 压入 **调用栈（Call Stack）**：

```javascript
function a() { b(); }
function b() { c(); }
function c() { console.log('在底'); }
a();
```

调用栈状态（自下而上）：

```
| c()    | <- 当前
| b()    |
| a()    |
| global |
```

`c` 执行完弹栈，回到 `b`，依次弹完。栈过深会 `Maximum call stack size exceeded`。

## 6. 作用域链 scope chain

函数能访问：自己的变量 → 外层函数的变量 → ... → 全局。这条链叫作用域链。

```javascript
const a = 1;
function outer() {
  const b = 2;
  function inner() {
    const c = 3;
    console.log(a, b, c);  // 1 2 3
  }
  inner();
}
outer();
```

`inner` 访问 `b` 时，自己没有 → 找 `outer` → 找到。这就是闭包的基础。

## 7. 闭包：JS 的灵魂

**一句话定义**：闭包 = 函数 + 它出生时所在的作用域。

只要一个函数被"带出"它的出生地（赋给变量、当回调、return 出去），它仍然记得外层变量。

```javascript
function makeCounter() {
  let count = 0;
  return function() {
    count++;
    return count;
  };
}
const c = makeCounter();
c();   // 1
c();   // 2
c();   // 3
// count 已经"逃出"了 makeCounter，但仍然存活在 c 的闭包里
```

> **类比 Python**：Python 的内层函数访问外层变量也是闭包，但要写 `nonlocal count` 才能修改。JS 不需要关键字，直接改。

## 8. 经典面试题：`for + setTimeout`

```javascript
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// 输出：3 3 3   ⚠️ 不是 0 1 2
```

**为什么**：`var` 是函数作用域，整个循环共用一个 `i`。3 个 setTimeout 回调都引用同一个 `i`，等到它们执行时 `i` 已经是 3。

**解法 1：用 `let`**（块级作用域，每次循环新建一个 `i`）

```javascript
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// 输出：0 1 2
```

**解法 2：IIFE 制造闭包**

```javascript
for (var i = 0; i < 3; i++) {
  (function(j) {
    setTimeout(() => console.log(j), 100);
  })(i);
}
```

## 9. 闭包用途

### 用途 1：私有变量（模块化）

```javascript
function createUser(name) {
  let _name = name;        // 私有，外面摸不到
  return {
    getName: () => _name,
    setName: (v) => { _name = v; }
  };
}
const u = createUser('Tom');
u.getName();          // 'Tom'
u._name;              // undefined  ✅ 真私有
u.setName('Jerry');
u.getName();          // 'Jerry'
```

### 用途 2：柯里化 curry

把 `f(a, b, c)` 变成 `f(a)(b)(c)`：

```javascript
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args);
    }
    return function(...args2) {
      return curried.apply(this, [...args, ...args2]);
    };
  };
}
const add = (a, b, c) => a + b + c;
const cAdd = curry(add);
cAdd(1)(2)(3);      // 6
cAdd(1, 2)(3);      // 6
cAdd(1)(2, 3);      // 6
```

### 用途 3：防抖与节流

```javascript
// 防抖：只在最后一次触发后 delay 毫秒执行
function debounce(fn, delay = 300) {
  let timer;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 节流：每 delay 毫秒最多执行一次
function throttle(fn, delay = 300) {
  let last = 0;
  return function(...args) {
    const now = Date.now();
    if (now - last >= delay) {
      last = now;
      fn.apply(this, args);
    }
  };
}
```

## 10. 内存泄漏与闭包

闭包持有外层变量，外层变量不会被 GC 回收。如果你在闭包里持有大对象（DOM、大数组）却长期不释放，就泄漏。

```javascript
function leak() {
  const huge = new Array(1e6).fill('x');
  return () => console.log(huge.length);
}
const f = leak();   // huge 永远活着
// 不用了就 f = null;
```

## 11. 实战练习

### 练习 1：实现 `once`（只执行一次）

```javascript
function once(fn) {
  let called = false, result;
  return function(...args) {
    if (!called) {
      called = true;
      result = fn.apply(this, args);
    }
    return result;
  };
}
const init = once(() => console.log('init!'));
init();   // init!
init();   // 啥也不打印
```

### 练习 2：实现私有计数器（带 reset）

```javascript
function counter() {
  let n = 0;
  return {
    inc: () => ++n,
    dec: () => --n,
    get: () => n,
    reset: () => { n = 0; }
  };
}
```

### 练习 3：闭包谜题
```javascript
function fn() {
  const arr = [];
  for (let i = 0; i < 3; i++) {
    arr.push(() => i);
  }
  return arr;
}
const [a, b, c] = fn();
console.log(a(), b(), c());   // 0 1 2  （let 每次新建）

function fn2() {
  const arr = [];
  for (var i = 0; i < 3; i++) {
    arr.push(() => i);
  }
  return arr;
}
const [x, y, z] = fn2();
console.log(x(), y(), z());   // 3 3 3  （var 共用）
```

## 常见错误

1. **以为 `var` 有块级作用域**：永远不要用 var
2. **闭包导致内存泄漏**：用完的引用置 null
3. **循环里 `var i` + 异步回调** → 改 `let`
4. **TDZ 报错却没意识到**：`let` 一定写在使用之前
