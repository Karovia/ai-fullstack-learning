# 命令行终端基础

> 图形界面是开车，命令行是骑摩托——后者风更大，但更快、更灵活，能钻进小巷。

## 一、为什么程序员要会用终端？🏍️

| 任务 | 图形界面 | 命令行 |
|------|----------|--------|
| 改 1 个文件名 | 双击改 | `mv a.txt b.txt` |
| 改 1000 个文件名 | 改到崩溃 | 一行命令搞定 |
| 远程操作服务器 | ❌ 不行 | ✅ SSH |
| 自动化任务 | 难 | 写脚本就行 |
| 给同事演示步骤 | 录屏 | 复制粘贴命令 |

90% 的开发工具（git、docker、npm、python）都是命令行优先的。**学会终端 = 解锁程序员的真正生产力**。

## 二、文件系统结构 🗂️

Linux/macOS 的目录是树状的，从 `/`（根目录）开始：

```
/                ← 根目录（一切的起点）
├── bin/         ← 系统命令（ls、cp 这些）
├── etc/         ← 系统配置
├── home/        ← 普通用户的家
│   └── alice/   ← 用户 alice 的家目录
├── usr/         ← 第三方软件
├── var/         ← 日志、缓存
└── tmp/         ← 临时文件
```

几个关键符号：

| 符号 | 含义 | 例子 |
|------|------|------|
| `/`  | 根目录 | `/etc/hosts` |
| `~`  | 当前用户家目录 | `~/Documents` |
| `.`  | 当前目录 | `./script.sh` |
| `..` | 上级目录 | `cd ..` |
| `-`  | 上一次所在目录 | `cd -` |

> macOS 上你的家目录是 `/Users/你的用户名`；Linux/WSL 上是 `/home/你的用户名`。`~` 是它的别名。

## 三、必会 30 命令场景手册 🛠️

### 导航三剑客

```bash
pwd              # 我现在在哪？(Print Working Directory)
ls               # 这个目录里有啥？
ls -la           # 包括隐藏文件 + 详细信息
cd /path/to/dir  # 去某个目录
cd ~             # 回家
cd -             # 回上一次
```

### 文件操作

```bash
mkdir my-dir              # 建目录
mkdir -p a/b/c            # 一次建多层
touch a.txt               # 建空文件 / 更新时间戳
cp a.txt b.txt            # 复制
cp -r dir1 dir2           # 递归复制目录
mv a.txt c.txt            # 重命名
mv a.txt /tmp/            # 移动
rm a.txt                  # 删文件
rm -rf dir                # 递归强删目录 ⚠️ 极度危险
```

> ⚠️ **`rm -rf /` 会删掉你的整个系统**。终端没有回收站，删了就没了。

### 查看内容

```bash
cat file.txt              # 整个倒出来
less file.txt             # 分页看（按 q 退出）
head -n 20 file.txt       # 前 20 行
tail -n 20 file.txt       # 后 20 行
tail -f log.txt           # 实时跟踪文件追加（看日志神器）
```

### 搜索

```bash
grep "hello" file.txt          # 在文件里找"hello"
grep -r "TODO" .               # 当前目录递归找
grep -i "Error" log.txt        # 不区分大小写
find . -name "*.py"            # 找所有 .py 文件
find . -type f -size +10M      # 大于 10MB 的文件
```

### 权限

```bash
chmod +x script.sh        # 加上可执行权限
chmod 755 file            # 数字写法
chown alice:alice file    # 改所有者
ls -l                     # 看权限：-rwxr-xr-x
```

权限三组字母：所有者 / 同组 / 其他人。`rwx` = 读写执行。

### 进程

```bash
ps aux              # 列出所有进程
ps aux | grep node  # 找 node 进程
top                 # 实时进程监视（q 退出）
htop                # 更好看的 top（需要装：brew install htop）
kill 1234           # 杀进程号 1234
kill -9 1234        # 强杀
```

### 网络

```bash
curl https://api.github.com         # 发 HTTP 请求
curl -O https://xx.com/file.zip     # 下载文件
wget https://xx.com/file.zip        # 也是下载
ping google.com                     # 测连通
ssh user@server.com                 # 远程登录
scp file.txt user@server:/tmp/      # 远程拷贝
```

### 压缩

```bash
tar -czf x.tar.gz dir/              # 打包压缩
tar -xzf x.tar.gz                   # 解压
zip -r x.zip dir/                   # zip 压缩
unzip x.zip                         # zip 解压
```

### 自我检查 ✅

跑下面这串练手：

```bash
cd ~
mkdir terminal-test
cd terminal-test
echo "hello world" > a.txt
cat a.txt
cp a.txt b.txt
ls -la
grep "hello" a.txt
cd ..
rm -rf terminal-test
```

## 四、管道 `|` 与重定向 `>` `>>` 🔌

### 重定向 —— 把输出存到文件

```bash
echo "hello" > a.txt        # 写入（覆盖）
echo "world" >> a.txt       # 追加
ls -la > files.txt          # 把 ls 的输出存进文件
command 2> error.log        # 把错误信息存进 error.log
command > out.log 2>&1      # 把正常输出和错误都存到 out.log
```

### 管道 —— 把上一个命令的输出，喂给下一个命令

```bash
ls | grep ".py"                    # 列出 .py 结尾的
cat log.txt | grep "ERROR" | wc -l # 统计 ERROR 行数
ps aux | grep python | head -5     # 前 5 个 python 进程
history | grep git                 # 在历史命令里找 git
```

类比：管道 `|` 就像工厂流水线，每道工序加工后传给下一道。

## 五、环境变量与 PATH 🛤️

### 什么是环境变量？

可以理解为系统的"全局配置项"。

```bash
echo $HOME             # 你的家目录
echo $USER             # 当前用户
echo $SHELL            # 当前 shell
echo $PATH             # 重点！见下方
```

### PATH 是什么？

PATH 是一串目录，**当你输入一个命令时，系统会按顺序在这些目录里找它**。

```bash
echo $PATH
# /usr/local/bin:/usr/bin:/bin:/usr/sbin:...
```

> 这就解释了为什么有时候 `command not found` —— 程序装在了 PATH 之外的地方。

### 怎么改 PATH？

编辑 `~/.zshrc`（或 `~/.bashrc`），加一行：

```bash
export PATH="$HOME/.local/bin:$PATH"
```

然后：

```bash
source ~/.zshrc       # 让改动立即生效
```

## 六、`~/.zshrc` 配置文件 ⚙️

`~/.zshrc` 是 Zsh 启动时自动执行的脚本，每次开终端都会跑一遍。

常见配置：

```bash
# 别名（让长命令变短）
alias ll='ls -la'
alias gs='git status'
alias gp='git pull'
alias ..='cd ..'

# 环境变量
export EDITOR='code --wait'
export PATH="$HOME/.local/bin:$PATH"

# 函数
mkcd() { mkdir -p "$1" && cd "$1"; }
```

改完 `source ~/.zshrc` 生效。

## 七、常用快捷键 ⌨️

| 快捷键 | 作用 |
|--------|------|
| `Tab` | 自动补全（**最重要的快捷键**） |
| `Ctrl+C` | 中断当前命令 |
| `Ctrl+D` | 退出当前 shell / 结束输入 |
| `Ctrl+L` | 清屏（等同 `clear`） |
| `Ctrl+R` | 搜索历史命令（再按一次 Ctrl+R 翻下一个） |
| `Ctrl+A` | 光标到行首 |
| `Ctrl+E` | 光标到行尾 |
| `Ctrl+U` | 删到行首 |
| `Ctrl+K` | 删到行尾 |
| `Ctrl+W` | 删一个单词 |
| `↑` / `↓` | 翻历史命令 |
| `!!` | 上一条命令 |
| `!$` | 上一条命令的最后一个参数 |

🔥 **Ctrl+R 一定要会**！它是你穿越历史命令的传送门。

## 八、常见报错解决方案 🚨

### `command not found`

参考第五节。多半是 PATH 没配好。

### `permission denied`

```bash
ls -l file.txt        # 看权限
chmod +x file.sh      # 加执行权限
sudo command          # 用管理员（谨慎）
```

### `No such file or directory`

路径写错了。检查：

```bash
pwd                   # 我在哪？
ls                    # 我能看到这个文件吗？
```

注意大小写——Linux/macOS **区分大小写**！

### 卡住了 / 不动了

按 `Ctrl+C` 中断。如果是某个程序卡死：

```bash
ps aux | grep 卡死的程序名
kill -9 进程号
```

### 不小心 `rm -rf` 删了重要文件

😱 **没有回收站**。如果在 git 仓库里、且之前 commit 过：

```bash
git checkout HEAD -- 文件名
```

否则只能去找数据恢复软件，但希望渺茫。**永远先备份再 `rm`！**
