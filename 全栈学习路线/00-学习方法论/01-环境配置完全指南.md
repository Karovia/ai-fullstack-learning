# 环境配置完全指南

> 工欲善其事，必先利其器。在敲第一行代码之前，我们先把"厨房"准备好。

## 一、什么是"开发环境"？🍳

想象你要成为一名大厨。在炒菜之前，你需要：

- 一个**厨房**（操作系统 + 终端）
- 一套**锅碗瓢盆**（编辑器、Git、包管理器）
- **食材库**（各种编程语言运行时，比如 Python、Node.js）
- 一个**菜谱本**（项目代码）

"开发环境"就是程序员的厨房。配置环境 = 把这些工具装好、摆好、调好。

⚠️ 新手最容易在这一步劝退自己。但只要按本文一步步来，**任何报错都有解**。遇到问题先深呼吸，再复制错误信息去搜索。

## 二、macOS 配置流程 🍎

### 2.1 安装 Homebrew（包管理器）

**什么是包管理器？** 想象成 macOS 的 App Store，但是命令行版。你说一句"我要装 Python"，它就帮你装好。

打开"终端"App（Spotlight 搜 `Terminal`），粘贴：

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

> 国内网络慢可换清华镜像（搜 "homebrew 清华镜像"）。

装完后**重启终端**，运行：

```bash
brew --version
```

看到版本号就成功了。

### 2.2 安装 iTerm2（更好用的终端）

```bash
brew install --cask iterm2
```

**为什么不用系统自带终端？** iTerm2 支持分屏、热键、配色更舒服，是 macOS 程序员的标配。

### 2.3 安装 Oh My Zsh（命令行美化）

```bash
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
```

装完后你的终端会变好看，并且支持智能补全。推荐主题：

```bash
# 编辑配置文件
nano ~/.zshrc
# 找到 ZSH_THEME，改成：
ZSH_THEME="agnoster"
```

保存（Ctrl+O，回车，Ctrl+X），然后：

```bash
source ~/.zshrc
```

### 2.4 自我检查 ✅

```bash
brew --version    # 应输出 Homebrew 4.x.x
echo $SHELL       # 应输出 /bin/zsh
which zsh         # 应输出 /bin/zsh 或 /opt/homebrew/bin/zsh
```

## 三、Windows 配置流程 🪟

⚠️ **强烈建议用 WSL2，不要直接在 Windows 原生环境开发。** 90% 的教程是为 Linux/macOS 写的，WSL2 让你在 Windows 里直接跑 Linux。

### 3.1 启用 WSL2

按 `Win 键` → 搜索 `PowerShell` → 右键"以管理员身份运行"，输入：

```powershell
wsl --install
```

这条命令会自动启用功能并装上 Ubuntu。装完**重启电脑**。

重启后会自动打开 Ubuntu 终端，让你设置用户名和密码。

> 💡 **密码不会显示**——这是正常现象，不是键盘坏了。盲打输入即可。

### 3.2 更新系统

```bash
sudo apt update && sudo apt upgrade -y
```

`sudo` = "以管理员身份执行"，会让你输密码（就是刚才设的那个）。

### 3.3 装常用工具

```bash
sudo apt install -y curl wget git build-essential
```

### 3.4 装 Windows Terminal（可选但强烈推荐）

到 Microsoft Store 搜 "Windows Terminal" 安装。它能在一个窗口里管理 PowerShell + WSL + CMD。

### 3.5 自我检查 ✅

```bash
lsb_release -a    # 应显示 Ubuntu 22.04 或更新
git --version     # 应显示 git version 2.x.x
```

## 四、安装 Python（用 uv，不要用 Anaconda）🐍

### 为什么不用 Anaconda？

| 方案 | 优点 | 缺点 |
|------|------|------|
| Anaconda | 一键装齐 | 庞大臃肿、慢、生态封闭 |
| 系统 Python | 自带 | 不能乱装包，可能搞坏系统 |
| **uv** | 极快、现代、Rust 写的 | 较新（但已稳定） |

uv 是 2024 年开始流行的新工具，速度比 pip 快 10-100 倍，是当前**业界推荐**。

### 安装 uv

**macOS / WSL 通用：**

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

重启终端后：

```bash
uv --version
```

### 用 uv 装 Python

```bash
uv python install 3.12
```

### 自我检查 ✅

```bash
uv python list           # 列出已装版本
uv run python --version  # 应输出 Python 3.12.x
```

## 五、安装 Node.js（用 nvm + pnpm）📦

### 为什么用 nvm？

不同项目可能要不同 Node 版本。nvm = Node Version Manager，可以一键切换。

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
```

**重启终端**，然后：

```bash
nvm install --lts    # 装最新长期支持版
nvm use --lts
node --version       # 应显示 v20.x.x 或更新
```

### 装 pnpm（比 npm 更快的包管理器）

```bash
npm install -g pnpm
pnpm --version
```

| 工具 | 速度 | 磁盘占用 |
|------|------|----------|
| npm  | 慢   | 巨大 |
| yarn | 中   | 大 |
| **pnpm** | **快** | **小** |

## 七、常见报错解决方案 🚨

### `command not found: xxx`

**原因：** 程序没装，或装了但不在 PATH 里。

**排查：**

```bash
which xxx          # 看路径
echo $PATH         # 看 PATH 包不包含
```

**解决：** 把工具的目录加到 `~/.zshrc` 或 `~/.bashrc`：

```bash
export PATH="$HOME/.local/bin:$PATH"
```

然后 `source ~/.zshrc`。

### `permission denied`

**原因：** 没权限。

**解决：** 加 `sudo` 重试，或检查文件权限 `ls -l 文件`，必要时 `chmod +x 文件`。

⚠️ **不要无脑 `sudo`！** 用户目录下的操作不应该需要 sudo。如果一个命令在你家目录都需要 sudo，说明前面操作错了。

### Homebrew 巨慢

换镜像。搜 "homebrew 中科大镜像" 跟着配。

### WSL 装好了打不开

PowerShell 管理员运行：

```powershell
wsl --update
wsl --shutdown
```

再重新打开。
