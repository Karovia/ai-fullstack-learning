# 第 0 章 · 如何高效学习编程（必读）

> 这一章不是技术，但比任何技术都重要。**没读完这章不要进 01 章**。

## 🎯 本章目标

- 理解为什么"看视频学不会编程"
- 建立你的学习节奏与反馈系统
- 配置好开发环境
- 学会 Git/GitHub 基本工作流
- 学会用 AI（Claude/Cursor）当老师而非保姆

## 1. 你为什么学不会？三个常见陷阱

| 陷阱 | 表现 | 解药 |
|------|------|------|
| **教程地狱** | 看了 N 门课，写不出 1 个自己的项目 | 70% 时间写代码，30% 看教程 |
| **复制粘贴幻觉** | 跟着抄能写，不看就空白 | 每个例子先关教程默写一遍 |
| **AI 依赖症** | 让 AI 直接给答案，不自己思考 | 先自己写 → 卡 15 分钟 → 再问 AI |

## 2. 学习循环（每节都按这个走）

```
读 → 仿 → 关掉教程默写 → 改造（加新功能）→ 写笔记教别人 → 一周后再回顾
 ↑                                                                ↓
 └─────────────── 复习不到位就回到"读" ─────────────────────────┘
```

## 3. 时间安排建议

零基础 → 源码级，**每天 3 小时是底线**。建议：
- 工作日：早起 1.5h（学新内容） + 通勤听播客 + 晚上 1.5h（写代码）
- 周末：6-8h 集中做项目，1h 写周报
- **每周必须 GitHub 有 5+ 天绿点**（commit 真实代码，不要刷）

## 4. 环境配置（本周完成）

### macOS / Linux 用户

```bash
# 1. Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 2. Git + GitHub CLI
brew install git gh
gh auth login

# 3. Python（uv 是 2025 后新标准）
curl -LsSf https://astral.sh/uv/install.sh | sh

# 4. Node.js
brew install nvm
nvm install --lts
npm i -g pnpm

# 5. VS Code
brew install --cask visual-studio-code
```

### Windows 用户

强烈建议 **WSL2 + Ubuntu**，然后按 Linux 教程走。原生 Windows 后期会撞很多坑。

### 必装 VS Code 插件

- Python
- Pylance

- Prettier
- GitLens
- Error Lens
- Tailwind CSS IntelliSense（后期用）
- Continue 或 Cursor（AI 辅助）

## 5. Git/GitHub 必会 10 个命令

```bash
git init
git clone <url>
git add .
git commit -m "feat: ..."
git push origin main
git pull
git branch feature/x
git checkout feature/x
git merge main
git log --oneline
```

约定式提交格式：`feat: / fix: / docs: / refactor: / test: / chore:`

详见：[Conventional Commits](https://www.conventionalcommits.org/zh-hans/)

## 6. 怎么用 AI 当老师（关键）

### ✅ 好的提问

> "我写了这段 Python 函数想反转链表，但运行报错。报错是 X。我猜是 Y 的问题，对吗？请只**指出方向**，不要直接给代码。"

### ❌ 差的提问

> "帮我写一个反转链表的函数。"

**铁律**：
- **第一次遇到的概念**，必须先自己读官方文档 30 分钟，再问 AI。
- AI 给的代码**先逐行解释一遍**给自己听，再粘贴。
- 项目代码必须能**关掉 AI 从头写一遍**才算掌握。

## 7. 反馈系统

| 频率 | 形式 | 在哪里 |

| 每章 | 章节项目 + 项目复盘 | [[_我的进度追踪/项目里程碑]] |

## 🚀 章节项目：建立你的"学习仪表盘"

### 需求
建立一个简单的网页（纯 HTML+CSS+JS，不用框架），展示你的学习进度。

### 技术栈
- HTML5 + CSS3 + 原生 JS
- 部署到 GitHub Pages

### 验收标准
- [ ] 展示 10 个章节列表 + 进度条
- [ ] 点击章节能展开看任务清单
- [ ] 进度数据存在 `localStorage`，刷新不丢
- [ ] 已部署到 GitHub Pages，能用 URL 访问
- [ ] 代码 commit 信息符合约定式提交规范

> 这个项目你现在做不出来很正常 —— 学完 02、03 章再回来重做，会有恍然大悟的感觉。先**完成它的最简版**（哪怕是丑的纯文本），重点是走通"写代码 → push → 部署 → 公网访问"全流程。
